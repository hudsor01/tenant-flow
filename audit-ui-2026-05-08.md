# TenantFlow Website Audit — 2026-05-08

**Source:** External UI audit of tenantflow.app, 2026-05-08
**Scope:** homepage, pricing, features, compare, about, blog, resources, contact, support, FAQ, security/terms/privacy, signup, mobile (375px breakpoint)
**Intended use:** v1.0 milestone scope. All corrections must align to canonical design tokens defined in `src/app/globals.css` (oklch color space, spacing/radius/shadow/duration/easing token system, typography scale).
**Status of investigation already done:**
- `src/components/sections/stats-showcase.tsx` already has correct values (5, 7, 500, 14). The "0" rendering is a `NumberTicker` animation/intersection-observer issue, not a data bug — counter starts at 0 and isn't animating to target.
- `/blog/[slug]/page.tsx` reads from a Supabase `blogs` table. Every post showing "Error Processing Blog" indicates bad DB rows (data-broken), not a component bug. Fix is data cleanup + content regeneration, not a code rewrite.
- `/signup` route does not exist in `src/app/`. Proxy redirects to `/login?redirect=%2Fsignup`; login redirects right back. Either add a real route or 301 to `/pricing`.
- `globals.css` is the canonical source for color/spacing/typography/radius/shadow/duration/easing tokens (`--color-*`, `--spacing-*`, `--text-*`, `--radius-*`, `--shadow-*`, `--duration-*`, `--ease-*`, `--color-chart-{1..5}`, `--color-{success,warning,info,destructive}`).

## Critical — Fix Before Any Marketing Spend

**1. The entire blog is broken.** Every single one of the ~70+ blog posts (12 pages of pagination) shows the title "Error Processing Blog" and the body text "Error: Could not extract content. Response keys: error". Hero images, categories, and dates render normally, but no actual content exists. Example URL: `/blog/error-1778151609106`. SEO disaster (Google indexing dozens of duplicate "Error Processing Blog" pages) + credibility kill.

**2. Homepage stat counters all display "0".** In the "Built for landlord workflows" section, the four stats render as: "0 Entity Branches", "0 Default Categories", "0 Bulk-Zip Cap", "0 Day Free Trial". Source data is correct (5, 7, 500, 14 in `stats-showcase.tsx`); the `NumberTicker` component is failing to animate to target.

**3. Pricing inconsistency between the Max plan card and the comparison table.** The Max pricing card (top of /pricing) shows "Custom" as the price, but the "Compare All Features" table directly below shows "Max — $199/mo", and the home-page features grid says "Max unlimited" with no price. The structured-data JSON-LD on the page also lists Max at $199. Pick one and use everywhere.

**4. Mobile layout broken at 375px.** Hero text "spreadsheet" overflows the viewport horizontally, the "Start Managing Properties" CTA button is cut off on the right, and there is no visible hamburger menu — the entire desktop nav (Features, Pricing, Compare, About, Resources, Sign In, Get Started) simply disappears, leaving mobile users with no navigation at all.

**5. /signup redirects to /login (in a loop).** Clicking any "Get Started" or "Start Free Trial" button funnels users to /pricing, but the standalone /signup route redirects to `/login?redirect=%2Fsignup` — circular redirect that prevents new account creation through that path.

**6. Footer/legal URLs return inconsistent results.** `/terms-of-service`, `/privacy-policy`, `/help-center`, `/rss-feed`, and `/signup` all redirect to login. The footer uses the working short paths (`/terms`, `/privacy`, `/help`, `/feed.xml`), but those long URLs are likely linked from emails, sitemaps, or external sources. Either alias both versions or 301 the long paths to the working ones.

## High Priority — Inconsistencies and Errors

**7. Contradictory headline messaging across pages.** Hero says "Ditch the spreadsheet" / "operations tool for property owners". About page says "Property management built for landlords" and "empower property managers". Meta description says "property administration software for owners and real estate investors". At least four different audiences named: property owners, landlords, property managers, real estate investors. Pick ONE primary persona.

**8. Wrong icon on "Multi-Property Dashboard" feature card.** Homepage features grid uses a back-arrow icon (←) on this card. Should be a dashboard, grid, or building icon (lucide-react).

**9. "Compare" nav item shows as the active page when on the homepage.** The Compare link in the top nav is underlined/bold even though the user is on `/`. Active-state highlight targeting wrong link.

**10. Date inconsistencies on legal pages.** Security Policy: "Last Updated: February 27, 2026". Terms of Service: "Last Updated: October 5, 2025". Footer copyright: "© 2026 TenantFlow". Either Terms is stale or Security is post-dated.

**11. The "Most Popular" badge on the Growth pricing card overlaps the card border** and looks like a layout bug.

**12. Inconsistent CTA button styling.** Primary blue "Start Free Trial" used most places, but: Starter plan CTA is solid black; "Talk to Sales" / "Contact Sales" buttons appear unstyled (white with no border in some places, with border in others); on Features page a duplicate "Start Free Trial" pill appears in top nav alongside "Get Started" — visual collision.

**13. Comparison table framing problems on /compare/buildium.** "ACH/Payment Processing" and "HOA Management" show a red ✗ for TenantFlow with the explanation "Landlord-only platform". Negative visual marker used to describe a positioning choice. Use a neutral icon or "By design" label.

**14. The "How did you hear about us?" dropdown on /contact defaults to "Sales Outreach".** Implies cold email. Default should be "Please select" or "Search engine".

**15. Pricing-page subhead copy error.** Under the Starter card it says "Ideal for property owners managing a few properties" with no period, then "/mo" appears on its own line. Awkward break/spacing.

**16. Annual toggle says "Save $158" but no annual prices are displayed without toggling.** After toggling, verify the math (Starter $29 × 12 = $348 / annual likely ~$290 = $58 savings; Growth $79 × 12 = $948). $158 doesn't match a single-plan calculation cleanly — show the math or per-plan savings.

**17. Resources nav dropdown link is `href="/#"`** (scrolls to top instead of opening or navigating). Does nothing useful for keyboard users.

**18. Footer copyright reads "© 2026 TenantFlow"** while current date is May 2026 — fine, but "Powered by Hudson Digital" on a B2B SaaS marketing site looks like a freelancer's signature. Remove for the campaign.

**19. Trusted Integrations row has visual inconsistency.** Stripe, Vercel, DocuSeal, and Resend logos appear sharp and dark; Supabase appears greyed out / faded. Either hover state stuck off, or one logo wasn't exported correctly.

**20. Duplicate comparison tables.** The "Why Landlords Choose TenantFlow" table appears identically on the homepage AND on the Features page. Either remove from one or differentiate them.

## Design-Token Alignment Constraints (apply to ALL fixes)

- Color: only use `--color-*` tokens defined in `globals.css` (oklch); never hex/rgb/named colors.
- Surfaces: `bg-background`, `bg-card`, `bg-muted` — never `bg-white` (breaks dark mode).
- Text: `text-foreground`, `text-muted-foreground` — never bare `text-muted` or hex.
- Status colors: `--color-{success,warning,info,destructive}` for status badges; never inline pinks/greens/oranges.
- Spacing: `--spacing-*` scale tokens.
- Radius: `--radius-{sm,md,lg,xl,2xl,full}` only.
- Shadow: `--shadow-{sm,md,lg,xl,2xl}` only.
- Typography: `--text-{display-*,title-*,base,lg,xl,sm,xs,caption,stat-*}` scale; never one-off font sizes.
- Animation: `--duration-*` + `--ease-*` tokens; never inline ms or custom cubic-beziers.
- Icons: `lucide-react` only (per CLAUDE.md zero-tolerance rule); never `@radix-ui/react-icons` or emojis.
