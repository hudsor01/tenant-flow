# TenantFlow v6.0 ŌĆö "Final Canonical Cleanup" Legacy-Remnant Audit

> Consolidated from 7 forensic finder reports (tenant-as-user, rent-facilitation, Stripe Connect, screening, dead DB objects, deferred fallow dead-code, docs/config/comments).
> Product = landlord-only property-management SaaS. Tenants are RECORDS, not users. Billing = landlord Stripe SUBSCRIPTION only. No rent facilitation, no Stripe Connect marketplace, no tenant portal, no automated screening.

---

## 1. Executive Summary

The 2026-04-18 `20260418140000_demolish_rent_and_tenant_portal.sql` migration (plus follow-ups `20260528202741`, `20260528204818`, `20260530015823`) already removed the heavy machinery: every rent-facilitation table (`rent_payments`, `rent_due`, `late_fees`, `payment_methods`, `payment_reminders`, `stripe_connected_accounts`), every facilitation/Connect function (`record_rent_payment`, `process_autopay_charges`, `calculate_late_fees`, `notify_n8n_rent_payment`, etc.), the legacy custom `stripe` schema, and `leases.stripe_connected_account_id`. **All four reviewers confirmed against prod: 0 surviving facilitation tables/functions/triggers/cron jobs.**

What remains is **orphaned application-layer fog** ŌĆö frontend UI, dead types, stale copy, and exactly **one** missed DB column ŌĆö that still references the demolished features. None of it touches the keep-side subscription billing (`stripe-checkout/-session`, `stripe-billing-portal`, `stripe-cancel-subscription`, `stripe-webhooks`, `users.subscription_*`).

**Risk profile:**
- **HIGH (customer-facing contradictions, ship soon):** 3 live UI surfaces actively promise demolished features ŌĆö the `/profile` "Payment Settings / receive payments from tenants" card, the `/financials` "Payouts" 404 dead-link, and onboarding "connecting Stripe for payouts, and inviting your first tenant" copy. These are the only HIGH-risk items.
- **MED (coupled DB + query change):** exactly **1 dead DB object** survives ŌĆö `properties.stripe_connected_account_id` (uuid, 0 non-null rows, no FK/index/RLS/RPC/trigger reader). It is still SELECTed live in `PROPERTY_SELECT_COLUMNS`, so the column drop and the select-string edit MUST ship in lockstep or property list queries 400. Plus the lease-activation `STRIPE_CONNECT_NOT_SETUP` / `STRIPE_VERIFICATION_INCOMPLETE` error gating, pending an INVESTIGATE on whether the activate RPC still emits those codes.
- **LOW (zero-consumer dead code):** ~20 dead TypeScript types/contracts (rent-facilitation, Connect, tenant-portal shapes), orphan mutation keys, a real `useCurrentUser` duplicate, two orphan `app_config` webhook-URL rows, a broken `seed.sql` insert, and stale "invite" wording.

**Scale:** ~38 distinct REMOVE items + ~9 INVESTIGATE items across 6 dimensions. The dead-code-export sweep (fallow) found 711 issues but only ~20 are product-boundary REMOVEs; 111 are KEEP-CONTRACT type surface and 28 of 29 "duplicate exports" are intentional ZodŌåötype co-location. The cleanup is **broad but shallow** ŌĆö mostly trivial deletions, one coordinated DB migration, and a careful copy rewrite.
