# TenantFlow Comprehensive Review — 2026-03-04

**8 parallel review agents** | **921 source files + 131 migrations + 13 Edge Functions**
**Total findings: 131** — 22 P0, 35 P1, 46 P2, 28 P3

## P0 — CRITICAL (22 findings)

### Security — Data Exfiltration
1. 12+ SECURITY DEFINER RPCs accept `p_user_id` without verifying `auth.uid()` — complete data exfiltration vector
2. `tenant-invitation-accept` unauthenticated — accepts arbitrary `authuser_id`
3. Error monitoring RPCs expose ALL user data (`get_error_summary`, `get_common_errors`, `get_error_prone_users`)
4. `activate_lease_with_pending_subscription` — no auth check, any user activates any lease
5. `sign_lease_and_check_activation` — any user signs any lease as either party

### Security — Auth/Middleware
6. Middleware may not be registered (`proxy.ts` exports `proxy` not `middleware`)
7. No role-based route enforcement — tenants access owner routes and vice versa
8. `AuthProvider` uses `getSession()` instead of `getUser()` — JWT not server-validated

### Financial — Payment Bugs
9. Potential 100x overcharge — `rent_due.amount * 100` when amount may already be in cents
10. `rent_due.status` never updated to `paid` after successful payment
11. Autopay broken — RLS blocks tenant lease UPDATE for `auto_pay_enabled`
12. `stripe-checkout-status` Edge Function doesn't exist (hooks invoke nonexistent function)
13. Payment display bug — webhook stores dollars, UI calls `formatCents()` dividing by 100 again

### Code Quality
14. Fake table cast `from('reports' as 'properties')` — queries wrong table
15. Missing env var validation in ALL 13 Edge Functions
16. Webhook inserts payment with empty FK refs (`tenant_id`/`lease_id` fallback to `''`)

### UX
17. `text-muted` used instead of `text-muted-foreground` across 69 files (150 occurrences)

### Testing/CI
18. No `next build` in CI
19. No coverage enforcement
20. Stale E2E webServer path
21. Zero tests for all 13 Edge Functions
22. Zero tests for 22 validation schemas + 19 utility files
