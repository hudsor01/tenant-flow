# v5.0 — AI Blog Content Engine (milestone scope, for approval)

**Status:** DRAFT scope (pending approval → formalize into REQUIREMENTS.md + ROADMAP.md, archive v4.0 first)
**Date:** 2026-06-07

## Core value
A **local-LLM (LM Studio, on the M5) n8n pipeline** that drafts **brand-positive, factually-grounded, E-E-A-T-credible** blog posts into the existing `n8n-blog-ingest` Edge Function as `status='in-review'` drafts for **human approval** — and uses that engine to execute the **SEO-01 reclaim** (republish the deleted high-impression slugs, then drop them from `blog-redirects.ts`). Quality bar: content that genuinely *helps landlords* and features TenantFlow naturally — NOT generic AI listicles (which Google's helpful-content system penalizes and which would worsen the very rankings we're reclaiming).

## Grounded constraints (verified 2026-06-07)
- **LLM host:** LM Studio on the Mac (`localhost:1234`, OpenAI-compatible `/v1`). Installed: `qwen3-coder-30b-a3b-instruct` (chat), `qwen3-embedding-0.6b` (embeddings), `qwen3-reranker-0.6b` (rerank), `nomic-embed-text-v1.5`. **No general/instruct chat model yet** — only the coder-instruct.
- **n8n** runs in Docker/colima on the M5 behind the `tenantflow-n8n` Cloudflare tunnel. **The container cannot reach the Mac host yet** (`host.docker.internal` unresolvable under colima) — must be wired.
- **Ingest contract:** `n8n-blog-ingest` EF requires HMAC-SHA256 (`x-n8n-signature`, key `N8N_WEBHOOK_SECRET`) + 9 validation gates; DB `validate_blog_post` trigger enforces slug format, the owner's `document_categories`-style blog category set, `canonical_url`, required NOT-NULL fields. Posts land `status='in-review'` → human gate before publish.
- **SEO-01 source:** ~126 deleted ghost slugs (top-10 by GSC impressions) in `src/lib/seo/blog-redirects.ts`; reclaim = publish a quality post at the slug, then delete its redirect entry (collision-guard test must stay green).

## Requirements (BLOG-xx)
- **BLOG-01** — n8n→LM Studio connectivity: wire the colima container to reach the Mac host (host-gateway or LAN IP); confirm `/v1/chat/completions` + `/v1/embeddings` reachable from n8n.
- **BLOG-02** — Generation model: pull a **general instruct** model into LM Studio for marketing prose (recommend `Qwen3-30B-A3B` general-instruct — same A3B speed, general-purpose; or Qwen2.5-32B-Instruct). Document choice + the coder-vs-general trade-off; coder-instruct is the fallback.
- **BLOG-03** — RAG grounding: a curated corpus of real TenantFlow facts (features, pricing from `pricing.ts`, positioning, landlord-domain expertise) embedded via `qwen3-embedding` into a retrievable store (pgvector in Supabase, or an n8n-reachable vector store); retrieval returns relevant context per topic so posts are accurate + distinctive (the real E-E-A-T lever).
- **BLOG-04** — Structured generation conforming to the ingest contract: LLM emits title/slug/excerpt/markdown-body/category/canonical_url that pass all 9 gates + `validate_blog_post`; deterministic post-format/validate step before send.
- **BLOG-05** — Delivery: HMAC-signed POST to `n8n-blog-ingest` → `status='in-review'` draft; Vercel deploy hook only on actual publish.
- **BLOG-06** — Quality + brand guardrails: brand-voice system prompt, E-E-A-T (Organization author "TenantFlow Team"), anti-hallucination (RAG-grounded facts only), a **self-critique / reranker pass** to reject thin or off-brand drafts before they reach in-review.
- **BLOG-07** — Human approval workflow: leverage `in-review` status; a review/approve surface (dashboard or n8n) so nothing publishes without you.
- **BLOG-08** — SEO-01 reclaim: topic queue seeded from the deleted high-impression slugs; generate at the exact original slug; on publish, auto-remove the entry from `blog-redirects.ts` (collision-guard test green). Closes the open v4.0 SEO-01 item.
- **BLOG-09** — Cadence + observability: schedule (e.g., a few/week), dedupe against existing/published slugs, execution monitoring + failure alerts (can reuse the new critical-error notify path).

## Key decisions still open
- **Model:** which general instruct model to pull (recommend Qwen3-30B-A3B general-instruct). The coder-instruct works but is dry for marketing.
- **Vector store:** pgvector in Supabase (keeps it in the existing stack, RLS-scoped) vs. a standalone store n8n hits. Recommend **pgvector** (already have Supabase; one less moving part).
- **Topic source priority:** SEO-01 reclaim slugs first (highest ROI — recovering ~10.9k impr/qtr) vs. net-new topics. Recommend **reclaim-first**.
