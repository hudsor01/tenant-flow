# v1.0 "Marketing Surface Honesty" — Resume Handoff (ARCHIVED)

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Hook bypass via `LEFTHOOK_EXCLUDE` or `--no-verify` | Discovered when a planner auto-tried `LEFTHOOK_EXCLUDE=lockfile-verify` to work around the sandbox `bun install` failure. Both are forbidden per project policy — they route around the user's deny rules. | blocking | When a commit is blocked by the sandbox `lockfile-verify` step (`bun install --frozen-lockfile` returns `PermissionDenied`), the ONLY sanctioned workaround is to run the `git commit` call with the command sandbox DISABLED so the full lefthook gate runs normally. Verify the lockfile is genuinely in sync first (it has been all session). |
| Code-fixer agents reporting `none_fixed` after applying edits | Phase 10 review's IN-01/IN-02 fixes — the fixer applied the edits to disk but mis-reported "no changes" so the followup PR #739 was needed. | advisory | After every code-fix sub-agent return, run `git status --short` and `git diff --stat` to verify what actually landed against what the agent reported. |
| Guessing TypeScript type-narrowing behavior without empirical verification | Phase 12 review cycle exposed a `NavHref` literal-narrowing claim that wasn't actually true. The fix required (a) writing a tiny TS probe to verify the actual TS 6.0 behavior of `as const satisfies` + `.flatMap`, and (b) loosening 3 consumer prop types from `NavItem[]` → `readonly NavItem[]`. | advisory | For TS type-narrowing claims in tests, write a deliberate-typo probe (`@ts-expect-error`-style) to prove the claim empirically before asserting it works. Engage `superpowers:systematic-debugging` after 2+ failed guesses. |
| Trusting GSD STATE.md / phase-directory tracking as ground truth | Several phases (04, 06, possibly 14) shipped to main without GSD recording SUMMARY.md files, causing `/gsd-progress` to mis-route. | advisory | When STATE.md or roadmap progress disagrees with git history, trust `git log main` and `gh pr list --state merged` as the source of truth. Use STATE.md as a working-state cache, not a milestone audit. |
