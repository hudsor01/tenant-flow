# Roadmap: TenantFlow v2.0 — Revenue Gates

**Defined:** 2026-04-19
**Target ship:** 2026-05-10 (3 weeks)
**Scope:** Prove the paywall-upgrade pattern by validating the shipped DocuSeal gate and adding one more: premium report exports. Kill everything else in the v2.0-audit backlog until there are real users to justify it.

## Why This Tight Scope

The 2026-04-16 revenue audit surfaced ~10 candidate features. Building all of them pre-product-market-fit is effort without payoff. For a product at this stage, the only reliable revenue levers are:

1. **Paywalls on features landlords already want** (forces Free → Growth upgrade)
2. **Cheap customer acquisition channels** (referral, SEO)

SEO already shipped in v1.6. Referrals are deferred until there are users worth referring. That leaves paywalls.

This milestone ships two paywalls and proves they convert. Everything else is deferred.

## Phases

- [ ] **Phase 45: DocuSeal gate prod validation** (3 days) — PR #595 paywall shipped but not traffic-validated. Confirm it's firing + converting.
- [ ] **Phase 46: Premium reports tier gate** (1 week) — paywall income-statement, cash-flow, year-end, and 1099 exports behind Growth tier.

## Phase 45: DocuSeal Gate Prod Validation

**Goal**: Prove the e-sign paywall is earning. Either it converts ≥5 Free→Growth upgrades in 7 days of active prod traffic, or we investigate why and fix the conversion funnel.

**Depends on**: PR #595 (shipped 2026-04-16)

**Requirements:**
- REQ-45-01: Stripe Checkout session metadata includes `source: 'esign_gate'` on any upgrade triggered by the DocuSeal paywall
- REQ-45-02: Analytics event (PostHog or equivalent) fires when the gate returns 402 to a Free-tier landlord
- REQ-45-03: Dashboard query showing `{gate_hits, upgrades_from_gate, conversion_rate}` for the last 30 days
- REQ-45-04: Playwright E2E test exercises the gate path end-to-end on a seeded Free-tier owner — asserts 402 response, CORS headers present, upgrade link in response body

**Battle-proven criterion:** ≥5 upgrades with `source: 'esign_gate'` in a rolling 7-day window after 2026-04-26 (allowing 1 week post-deploy). If below threshold, investigate funnel dropoff and either fix or cut the feature.

## Phase 46: Premium Reports Tier Gate

**Goal**: Year-end / 1099 / tax-doc exports require Growth tier. Landlords hit this in Jan/April tax season — a predictable upgrade trigger.

**Depends on**: Phase 45 proves the paywall pattern works.

**Requirements:**
- REQ-46-01: Report export endpoints (`export-report` Edge Function) check entitlement before generating. Reuse `checkESignEntitlement`-style helper extracted to `_shared/tier-gate.ts`
- REQ-46-02: Gated report types: `income_statement`, `cash_flow`, `year_end_summary`, `1099_vendor`. Free-tier users see a CTA on the reports page instead of a download button
- REQ-46-03: Upgrade CTA links to `/billing/plans?source=reports_gate` so the upgrade is attributable
- REQ-46-04: Playwright E2E: Free-tier owner clicks "Export 1099" → sees upgrade paywall, not a download
- REQ-46-05: Stripe Checkout metadata includes `source: 'reports_gate'` when the upgrade happens
- REQ-46-06: Free-tier still has basic reports (property summary, maintenance log) so the product is usable without upgrading

**Battle-proven criterion:** ≥10 Free→Growth upgrades with `source: 'reports_gate'` in 14 days post-deploy.

## Explicitly Deferred (audit backlog, not this milestone)

- Referral / affiliate program — defer until there's a user base to refer from
- API access / developer tier — requires devs as customers, niche at launch
- Accountant portal — nice-to-have, doesn't drive new revenue
- Mailing service (Lob) — niche, high eng cost, small margin
- Lemonade insurance referrals — small affiliate commission, cool-off period
- Tenant screening (TransUnion) — FCRA compliance is a real cost center
- Landlord credit reporting (RentBureau) — vendor onboarding, small initial audience
- Vendor marketplace — needs network effects that don't exist yet

Reopen any of these only after hitting a real engagement signal (user requests, churn analysis, etc.). Not on a schedule.

## Sequencing Rules

1. Phase 45 must hit its battle-proven criterion before Phase 46 gets built — if the first paywall doesn't convert, the second probably won't either and we should investigate WHY rather than add more gates
2. Both phases ship behind feature flag for 48h post-deploy before declared live
3. Kill criteria: if Phase 45 fails to hit 5 upgrades even after 30 days of traffic, rethink the paywall pattern before shipping Phase 46

## Revenue Target

A landlord on Growth pays $29/mo recurring. If this milestone produces:
- 5 upgrades from e-sign gate × $29 = $145 MRR
- 10 upgrades from reports gate × $29 = $290 MRR
- **Total: ~$435 MRR from ~15 upgrades**

Modest, but the point is pattern validation — if these work, later gates (premium imports, multi-user seats, etc.) follow the same template. If they don't, we stop throwing eng time at paywalls and rethink the product surface.
