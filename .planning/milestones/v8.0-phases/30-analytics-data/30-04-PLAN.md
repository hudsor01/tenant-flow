<objective>
Two cache-invalidation correctness bugs. PROP-01: the property-detail units table reads `unitQueries.byProperty(id)` = `['units','by-property',id]`, but the three unit mutations in `use-unit.ts` invalidate `unitQueries.lists()` = `['units','list']`, a sibling branch that does NOT prefix-match `by-property` — so the table stays stale after create/edit/delete. PROP-02: `useUpdatePropertyMutation` invalidates `ownerDashboardKeys.analytics.stats()` = `['owner-dashboard','analytics','stats']`, a key nothing ever queries; the dashboard reads `analytics.pageData()`, so a rename never refreshes it (the sibling create/delete property mutations correctly use `ownerDashboardKeys.all`).

Purpose: Unit mutations refresh the property-detail units table; property updates refresh the dashboard.
Output: Two one-line-family invalidation fixes.
</objective>
