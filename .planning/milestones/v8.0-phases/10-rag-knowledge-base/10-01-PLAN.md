    1. `create extension if not exists vector with schema extensions;` (pgvector is available-not-installed per the live probe in 10-CONTEXT.md).
    2. `create table if not exists public.blog_rag_chunks` with columns exactly: `id uuid primary key default gen_random_uuid()`, `content text not null`, `source text`, `metadata jsonb not null default '{}'::jsonb`, `embedding extensions.vector(1024)`, `created_at timestamptz not null default now()`, `updated_at timestamptz not null default now()`.     3. Cosine ANN index: `create index if not exists blog_rag_chunks_embedding_hnsw on public.blog_rag_chunks using hnsw (embedding extensions.vector_cosine_ops);`. If the prod PG build rejects `hnsw` at apply time (Task 2), fall back to `ivfflat (embedding extensions.vector_cosine_ops) with (lists = 100)` and note the fallback in the SUMMARY. Use cosine ops to match the RPC's `<=>` operator.
    4. Attach the canonical updated_at trigger: `create trigger set_blog_rag_chunks_updated_at before update on public.blog_rag_chunks for each row execute function public.set_updated_at();` — reuse the existing `set_updated_at()` function, never redefine it (Schema Conventions rule).
    5. `alter table public.blog_rag_chunks enable row level security;` Company knowledge, not per-owner data, so add NO owner-scoped SELECT policy — reads go exclusively through the SECURITY DEFINER RPC and writes are service-role (which bypasses RLS). Do NOT add a `FOR ALL` policy; do NOT grant table-level SELECT to authenticated/anon.
    6. The match RPC: `create or replace function public.match_blog_rag_chunks(query_embedding extensions.vector(1024), match_count int default 6) returns table (id uuid, content text, source text, metadata jsonb, similarity float) language sql stable security definer set search_path to 'public, extensions'` returning `select c.id, c.content, c.source, c.metadata, 1 - (c.embedding <=> query_embedding) as similarity from public.blog_rag_chunks c where c.embedding is not null order by c.embedding <=> query_embedding limit greatest(match_count, 1)`. Use `1 - cosine_distance` so similarity is higher-is-better. Include `extensions` in search_path so the vector operators resolve. Clamp limit with `greatest(match_count, 1)` to reject a zero/negative count.
    7. Lock down grants per the threat model: `revoke all on function public.match_blog_rag_chunks(extensions.vector, int) from public;` then `grant execute on function public.match_blog_rag_chunks(extensions.vector, int) to authenticated, service_role;`. NEVER grant to `anon` (avoids free embedding-compute / scraping abuse).

    Add a header comment block (like the precedent migration) stating: BLOG-03, embedding dim 1024 = qwen3-embedding, company-knowledge (no owner scope), reads via RPC only, no anon grant. No PostgreSQL ENUMs anywhere. No commented-out SQL.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| frontend/anon → DB | Untrusted callers must not reach the corpus or the embedding-search compute |
| authenticated client → match RPC | Authenticated callers may query retrieval; no per-owner data crosses (company facts) |
| indexer (service-role) → blog_rag_chunks | Trusted write path; service-role key never reaches the frontend |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Elevation of Privilege | match_blog_rag_chunks RPC | mitigate | SECURITY DEFINER with `set search_path to 'public, extensions'`; `revoke all from public`; grant execute only to authenticated + service_role; never anon |
| T-10-02 | Denial of Service | embedding-search compute via RPC | mitigate | No anon grant prevents unauthenticated free vector-search abuse; `match_count` clamped with `greatest(match_count, 1)` to reject zero/negative |
| T-10-03 | Information Disclosure | blog_rag_chunks table | accept | Contents are public company marketing facts (sourced from public/llms-full.txt); RLS enabled + no table-level read grant means reads only via the gated RPC; no PII present |
| T-10-04 | Tampering | corpus rows | mitigate | RLS enabled with no INSERT/UPDATE policy for authenticated/anon; only service-role (indexer) can write, and the service-role key stays out of the frontend bundle |
| T-10-SC | Tampering | npm/bun installs | mitigate | This plan adds NO new package-manager installs (pgvector is a DB extension, not an npm package); no supply-chain surface in 10-01 |
</threat_model>
