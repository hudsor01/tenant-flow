<objective>
Harden the shared `NumberTicker` primitive with an INTERNAL JS reduced-motion guard (POLISH-08 / D-04). A CSS `@media (prefers-reduced-motion)` rule cannot stop a JS `requestAnimationFrame` chain — the primitive must read the preference in JS and skip the rAF loop, snapping `displayValue` to the final `to`. The change is strictly additive: the new behavior triggers ONLY under `prefers-reduced-motion: reduce`; the motion-on (default) path is byte-for-byte unchanged. `NumberTicker` has 16 app-wide consumers, so the existing 6-test suite must stay green and a reduced-motion branch test is added. Charts (`isAnimationActive={!reducedMotion}`) and `BlurFade` are already guarded — verify only, do not edit.

Purpose: Success criterion #5 (reduced-motion users see no `NumberTicker`, `BlurFade`, or chart-transition animations). The `KpiNumberTicker` wrapper in `kpi-bento-row.tsx` already guards at the dashboard call site, but the underlying primitive does not — so every other NumberTicker consumer still animates under reduced motion until this lands.
Output: `number-ticker.tsx` honors `useReducedMotion()` internally; one new branch test; chart/BlurFade guards confirmed.
</objective>
