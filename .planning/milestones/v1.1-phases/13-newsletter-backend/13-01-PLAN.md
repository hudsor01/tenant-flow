  <name>Task 1: Create newsletter-subscribe Edge Function</name>
  <files>supabase/functions/newsletter-subscribe/index.ts</files>
  <action>
Create the Edge Function following the established unauthenticated rate-limited pattern from `stripe-checkout-session/index.ts`.

**Structure (in this exact order):**
1. Imports: `getCorsHeaders`, `handleCorsOptions` from `../_shared/cors.ts`, `validateEnv` from `../_shared/env.ts`, `errorResponse` from `../_shared/errors.ts`, `rateLimit` from `../_shared/rate-limit.ts`
2. Constants: `EMAIL_REGEX = /^[^\s@]+@[^\s@]+\.[^\s@]+$/`, `SEGMENT_NAME = 'Newsletter'`, `RESEND_BASE = 'https://api.resend.com'`
3. Module-level cache: `let cachedSegmentId: string | null = null` (isolate-cached like `rate-limit.ts` pattern)
4. Helper function: `getOrCreateSegmentId(apiKey: string): Promise<string>` -- lists segments via `GET /segments`, finds existing "Newsletter" segment, creates via `POST /segments` if not found, handles race condition by re-listing on creation failure. Caches result in `cachedSegmentId`.
5. `Deno.serve(async (req: Request) => { ... })` handler

**Handler flow:**
1. `handleCorsOptions(req)` -- return if OPTIONS
2. Reject non-POST with 405 (plain text "Method Not Allowed" with CORS headers)
3. `rateLimit(req, { maxRequests: 5, windowMs: 60_000, prefix: 'newsletter' })` -- return if rate limited (NEWS-02)
4. `validateEnv({ required: ['RESEND_API_KEY'], optional: ['FRONTEND_URL', 'UPSTASH_REDIS_REST_URL', 'UPSTASH_REDIS_REST_TOKEN'] })`
5. Parse body, extract `email` as trimmed lowercase string
6. Validate email with `EMAIL_REGEX` -- return 400 `{ error: 'Valid email is required' }` if invalid (NEWS-02)
7. Call `getOrCreateSegmentId(apiKey)` to get segment ID
8. `POST https://api.resend.com/contacts` with `{ email, segments: [segmentId] }` (NEWS-01)
9. **Locked decision: always return 200 `{ success: true }` regardless of Resend response.** If Resend returns non-2xx (likely duplicate), log a structured warning with `console.error` (including status, body, email domain -- never the full email). Do NOT surface Resend errors to caller.
10. Catch block: use `errorResponse(req, 500, err, { action: 'newsletter_subscribe' })` for unexpected errors

**Resend API calls (raw fetch, NOT SDK -- per project convention):**
- `GET https://api.resend.com/segments` -- headers: `Authorization: Bearer ${apiKey}`
- `POST https://api.resend.com/segments` -- body: `{ name: 'Newsletter' }`
- `POST https://api.resend.com/contacts` -- body: `{ email, segments: [segmentId] }`

All Resend responses typed with inline type annotations (e.g., `as { data: Array<{ id: string; name: string }> }`). Do NOT create separate type files for Resend API shapes -- they're internal implementation detail.

**File header comment** (match stripe-checkout-session pattern):
```
// Newsletter Subscribe Edge Function
// NEWS-01: Creates contact in Resend via Contacts API with segment association.
// NEWS-02: Rate limited at 5 req/min per IP. Email validated server-side.
// Unauthenticated by design -- newsletter signup is public.
// Duplicates return success silently (locked decision).
//
// POST { email: string }
// -> 200 { success: true }
// -> 400 { error: 'Valid email is required' }
// -> 429 { error: 'Too many requests' }
// -> 500 { error: 'An error occurred' }
```

**Do NOT:**
- Import the Resend Node SDK (Node-only, not Deno compatible)
- Use deprecated Audiences API (`/audiences/*` endpoints)
- Create a Supabase client (no database interaction in this function)
- Require auth/Bearer token (public endpoint)
- Expose Resend error details to the caller
- Log the full email address (log only the domain for observability)
