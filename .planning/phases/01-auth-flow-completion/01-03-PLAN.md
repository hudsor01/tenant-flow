# 61-03: Google OAuth Routing

## Goal
Google OAuth users are assigned the correct `user_type` and routed to the matching dashboard. First-time Google OAuth users see a role selection screen. If a pending tenant invitation exists for their email, it auto-links.

## Current State
- Google button exists on login page (GoogleButton component)
- Login page calls `signInWithOAuth({ provider: 'google' })` with redirect to `/auth/callback`
- Callback route at `src/app/auth/callback/route.ts` exchanges code, reads `user_type` from `app_metadata`, routes TENANT to /tenant, else to /dashboard
- `ensure_public_user_for_auth` trigger creates public.users row on auth insert, defaults `user_type` to 'TENANT' when not in metadata
- `custom_access_token_hook` reads user_type from auth > public > user_metadata > defaults to 'OWNER'
- `sync_user_type_to_auth` trigger syncs public.users.user_type changes to auth.users.raw_app_meta_data
- No signup page exists in (auth) route group - owners sign up via pricing/checkout flow
- Accept-invite page handles tenant signups

## Key Issue
When a user signs in with Google for the first time:
1. Supabase creates an auth.users entry
2. `ensure_public_user_for_auth` trigger creates public.users with `user_type = 'TENANT'` (since no user_type in metadata)
3. `custom_access_token_hook` will read 'TENANT' from public.users
4. The user gets routed to /tenant dashboard but they might be an owner

This needs a role selection screen for first-time Google OAuth users.

### Task 2: Create role selection page

### Task 3: Auto-link pending tenant invitations
**File**: `apps/frontend/src/app/auth/callback/route.ts`
- After exchanging code for session, check if user_type is 'PENDING' or not set
- Query `tenant_invitations` for pending invitations matching the user's email
- If found: call the `tenant-invitation-accept` Edge Function with the invitation code and user ID
- Set user_type to 'TENANT', refresh session, route to /tenant
- If no invitation found: redirect to `/auth/select-role`

### Task 4: Update callback for role-based routing
**File**: `apps/frontend/src/app/auth/callback/route.ts`
- After session exchange, check user_type from app_metadata
- If 'PENDING' or missing: check for pending invitations (Task 3), then redirect to /auth/select-role
- If 'TENANT': redirect to /tenant
- If 'OWNER': redirect to /dashboard
- Add Google OAuth button to accept-invite hero or separate signup page (not needed - already on login)

## Acceptance Criteria
- Google OAuth first-time user: shown role selection screen
- Selecting "Property Owner": user_type set to OWNER, routed to /dashboard
- Selecting "Tenant": user_type set to TENANT, routed to /tenant
- Google OAuth user with pending invitation: auto-linked as tenant, routed to /tenant
- Returning Google OAuth users: routed directly to their dashboard
- PENDING users cannot access /dashboard or /tenant until selecting a role
