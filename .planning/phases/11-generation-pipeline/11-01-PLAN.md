<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| n8n (laptop) -> ingest EF (public) | Outbound POST over the internet; HMAC is the sole auth boundary (no Bearer, no RLS — EF inserts service-role) |
| n8n -> Supabase REST RPC | SUPABASE_SECRET_KEY (service-role-equiv) used server-side from the laptop only |
| n8n -> LM Studio | localhost only (no network exposure) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Spoofing | ingest EF POST | mitigate | HMAC-SHA256 over exact raw body with N8N_WEBHOOK_SECRET == the EF's secret; a 401 surfaces misalignment (Task 4 is the objective gate) |
| T-11-02 | Information disclosure | HMAC secret / SUPABASE_SECRET_KEY | mitigate | Never log either value in any Code/HTTP node; SUPABASE_SECRET_KEY lives only in an n8n credential (server-side), never reaches a browser |
| T-11-03 | Tampering | signed body integrity | mitigate | POST sends the byte-identical string that was HMAC'd (no re-serialization between sign and send) |
| T-11-04 | Elevation of privilege | RAG RPC access | accept | match_blog_rag_chunks is NOT granted to anon (verified Phase 10); SUPABASE_SECRET_KEY is authorized + server-side only |
| T-11-SC | Tampering | npm/n8n node installs | mitigate | No new packages installed; workflow uses built-in n8n nodes (httpRequest, code, set) + node crypto only |
</threat_model>
