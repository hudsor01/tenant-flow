   - From inspections: `update public.documents d set owner_user_id = i.owner_user_id from public.inspections i where d.entity_type = 'inspection' and d.entity_id = i.id and d.owner_user_id is null;`

4. **Rewrite RLS policies** -- drop all existing document policies and recreate using direct owner_user_id check. Per rls-policies skill: one policy per operation per role, consolidated owner+tenant in single SELECT policy, `(select auth.uid())` subselect pattern.

   SELECT policy (owner sees own docs, tenant sees docs for their leases):
   ```sql
   create policy "documents_select" on public.documents
     for select to authenticated
     using (
       owner_user_id = (select auth.uid())
       or entity_id in (
         select lt.lease_id from public.lease_tenants lt
         join public.tenants t on t.id = lt.tenant_id
         where t.user_id = (select auth.uid())
       )
     );
   ```

   NOTE: Tenant access intentionally limited to lease-type documents only. Documents with entity_type of inspection, maintenance_request, or property are owner-managed -- tenants access those entities through their lease relationship but do not need direct document access for them. The backfill populates owner_user_id from all 4 entity types for owner RLS, but the tenant SELECT path only joins through lease_tenants. This is by design, not an oversight. Add a SQL comment in the migration explaining this decision.

   INSERT policy (owner only):
   ```sql
   create policy "documents_insert_owner" on public.documents
     for insert to authenticated
     with check (owner_user_id = (select auth.uid()));
   ```

   UPDATE policy (owner only):
   ```sql
   create policy "documents_update_owner" on public.documents
     for update to authenticated
     using (owner_user_id = (select auth.uid()))
     with check (owner_user_id = (select auth.uid()));
   ```

   DELETE policy (owner only):
   ```sql
   create policy "documents_delete_owner" on public.documents
     for delete to authenticated
     using (owner_user_id = (select auth.uid()));
   ```

IMPORTANT: Before dropping existing policies, query the database to find exact policy names on the documents table. Use `select policyname from pg_policies where tablename = 'documents'` as a reference comment. Drop each existing policy by its actual name.
