1. **Unschedule the existing inline cron job:**
   ```sql
   select cron.unschedule('expire-leases');
   ```
   Wrap in a DO block with exception handling in case the job doesn't exist.

2. **Create the named function** following the established pattern from `calculate_late_fees()`:
   ```sql
   create or replace function public.expire_leases()
   returns void
   language plpgsql
   security definer
   set search_path = public
   as $$
   declare
     v_lease record;
     v_expired_count integer := 0;
   begin
     for v_lease in
       select id, owner_user_id
       from public.leases
       where lease_status = 'active'
         and end_date < current_date
       for update skip locked
     loop
       update public.leases
       set lease_status = 'expired', updated_at = now()
       where id = v_lease.id;

       -- notification for owner (per CONTEXT.md decision)
       insert into public.notifications (user_id, notification_type, entity_type, entity_id, title, message)
       values (v_lease.owner_user_id, 'lease_expired', 'lease', v_lease.id,
               'Lease Expired', 'A lease has expired and needs attention.');

       v_expired_count := v_expired_count + 1;
     end loop;

     raise notice 'expire_leases: expired % leases', v_expired_count;
   end;
   $$;
   ```

   IMPORTANT: Check the actual `notifications` table columns first -- verify column names (user_id, notification_type, entity_type, entity_id, title, message) match the schema. Look at existing notification inserts in the codebase for the correct column pattern. Also check if `notification_type` has a CHECK constraint -- if so, either use an existing type value or add 'lease_expired' to the constraint.

   Also check `leases.lease_status` valid values -- verify 'expired' is acceptable. Look for CHECK constraints on that column.

3. **Reschedule with the named function:**

   );
