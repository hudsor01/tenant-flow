# 64-02: Frontend — Autopay Toggle + Payment Settings

## Overview
Wire up the existing autopay page with real mutations that enable/disable autopay by updating the lease's `auto_pay_enabled` flag and `autopay_payment_method_id`. Update the autopay status query to read from the lease record directly.

## Steps

### 1. Update autopay status query
**File:** `apps/frontend/src/hooks/api/use-tenant-portal.ts`

Change the `autopay` query to read `auto_pay_enabled` and `autopay_payment_method_id` from the lease:
- Query: `leases.select('id, auto_pay_enabled, autopay_payment_method_id, rent_amount, payment_day, ...')`
- Return `autopayEnabled: lease.auto_pay_enabled`
- Return the next payment date calculated from `payment_day` and current month
- Return the payment method details by joining to `payment_methods` where `stripe_payment_method_id = autopay_payment_method_id`

### 2. Implement real setup autopay mutation
**File:** `apps/frontend/src/hooks/api/use-tenant-portal.ts`

Replace the stub `useTenantPortalSetupAutopayMutation`:
- Validate payment method is selected
- Get the `stripe_payment_method_id` from the selected `payment_methods` record
- Update lease: `auto_pay_enabled = true`, `autopay_payment_method_id = stripe_pm_id`
- Invalidate autopay query

### 3. Implement real cancel autopay mutation
**File:** `apps/frontend/src/hooks/api/use-tenant-portal.ts`

Replace the stub `useTenantPortalCancelAutopayMutation`:
- Update lease: `auto_pay_enabled = false`, `autopay_payment_method_id = null`
- Invalidate autopay query

### 4. Update autopay page
**File:** `apps/frontend/src/app/(tenant)/tenant/payments/autopay/page.tsx`

The page is already well-structured. Changes:
- Show the current autopay payment method (card brand + last 4) when enabled
- Show next payment date when enabled
- Ensure the toggle uses the real mutations
- Add confirmation when disabling (to prevent accidental toggles)

### 5. Update existing tests
**File:** `apps/frontend/src/app/(tenant)/tenant/payments/__tests__/payments-page-autopay-cta.test.tsx`

Update mock data to use `autopayEnabled` from the new query structure (should already work since the interface is unchanged).

## Verification
- Tenant can toggle autopay on from the autopay settings page
- Toggling on saves `auto_pay_enabled = true` and `autopay_payment_method_id` to the lease
- Toggling off clears both fields
- Status query reflects current state correctly
- Tests pass
