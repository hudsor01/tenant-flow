    Rewrite unit-keys.ts queryFn functions to use supabase.from() directly. Remove all apiRequest calls.

    Imports to add:
    - import { createClient } from '#lib/supabase/client'
    - import { handlePostgrestError } from '#lib/postgrest-error-handler'
    - REMOVE apiRequest import

    Migration for each queryOptions:

    1. unitQueries.list(filters?):
       - queryFn: const supabase = createClient()
       - let q = supabase.from('units').select('id, property_id, owner_user_id, unit_number, floor, bedrooms, bathrooms, square_feet, rent_amount, deposit_amount, status, created_at, updated_at', { count: 'exact' }).order('created_at', { ascending: false })
       - Apply .neq('status', 'inactive') by default unless filters.status is specified
       - Apply .eq('property_id', filters.property_id) if provided
       - Apply .eq('status', filters.status) if provided
       - Apply .ilike('unit_number', `%${filters.search}%`) if search provided
       - Apply .range(filters.offset ?? 0, (filters.offset ?? 0) + (filters.limit ?? 50) - 1)
       - const { data, error, count } = await q; if (error) handlePostgrestError(error, 'units')
       - Return { data: data ?? [], total: count ?? 0, limit: filters?.limit ?? 50, offset: filters?.offset ?? 0 }

    2. unitQueries.listByProperty(property_id):
       - queryFn: const supabase = createClient()
       - const { data, error } = await supabase.from('units').select('id, property_id, owner_user_id, unit_number, floor, bedrooms, bathrooms, square_feet, rent_amount, deposit_amount, status, created_at, updated_at').eq('property_id', property_id).neq('status', 'inactive').order('unit_number', { ascending: true })
       - if (error) handlePostgrestError(error, 'units')
       - return data ?? []

    3. unitQueries.detail(id):
       - queryFn: const supabase = createClient()
       - const { data, error } = await supabase.from('units').select('id, property_id, owner_user_id, unit_number, floor, bedrooms, bathrooms, square_feet, rent_amount, deposit_amount, status, created_at, updated_at').eq('id', id).single()
       - if (error) handlePostgrestError(error, 'units')
       - return data

    4. unitQueries.byProperty(property_id):
       - Same as listByProperty but returns Unit[] array (not paginated)
       - queryFn: fetch units where property_id = property_id and status != 'inactive'

    5. unitQueries.stats():
       - This called /api/v1/units/stats
       - Replace with: supabase.rpc('get_unit_stats') if it exists in supabase.ts types
       - Otherwise use aggregation: select count by status from units, return { total, available, occupied, maintenance }
       - Cast to UnitStats type

    - REMOVE useUser import if it was only used to get owner_user_id for NestJS calls (RLS handles this)

    Migrate each mutationFn:

    1. useCreateUnitMutation — mutationFn (data: UnitInput):
       - const supabase = createClient()
       - Get owner_user_id from RLS context — include it if the DB INSERT requires it
       - If needed: const { data: { user } } = await supabase.auth.getUser()
       - const { data: created, error } = await supabase.from('units').insert({ ...data, owner_user_id: user?.id }).select().single()
       - if (error) handlePostgrestError(error, 'units')
       - return created as Unit
       - Remove the owner_user_id: user?.id inclusion if RLS+default handles it automatically

    2. useUpdateUnitMutation — mutationFn ({ id, data, version? }):
       - const supabase = createClient()
       - const updatePayload = version ? { ...data, version } : { ...data }
       - const { data: updated, error } = await supabase.from('units').update(updatePayload).eq('id', id).select().single()
       - if (error) handlePostgrestError(error, 'units')
       - return updated as Unit

    3. useDeleteUnitMutation — mutationFn (id: string):
       - Soft-delete: set status = 'inactive'
       - const supabase = createClient()
       - const { error } = await supabase.from('units').update({ status: 'inactive' }).eq('id', id)
       - if (error) handlePostgrestError(error, 'units')
