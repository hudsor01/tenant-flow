    Create apps/frontend/src/lib/postgrest-error-handler.ts with a single exported function.

    The function signature: handlePostgrestError(error: PostgrestError, domain: string): never

    Implementation:
    1. Import PostgrestError from '@supabase/supabase-js'
    2. Import toast from 'sonner'
    3. Import * as Sentry from '@sentry/nextjs' (Sentry is already configured in the project)
    4. Map PostgrestError codes to user-friendly messages:
       - '23505' (unique violation) → 'This record already exists'
       - '23503' (foreign key violation) → 'Cannot complete: related record is in use'
       - '42501' (insufficient privilege / RLS block) → 'You do not have permission for this action'
       - PGRST116 (no rows returned) → 'Record not found'
       - Default fallback → error.message
    5. Show toast.error with domain context: e.g. "Failed to update property: Record not found"
    6. Capture to Sentry with full PostgrestError context:
       Sentry.captureException(new Error(`PostgREST error in ${domain}: ${error.message}`), {
         extra: { code: error.code, details: error.details, hint: error.hint, domain }
       })
    7. Throw the original error so TanStack Query registers it as failed: throw error

    The function returns never because it always throws.

    File must NOT be a barrel file — only export handlePostgrestError.
    Do NOT use `any` — use PostgrestError type from supabase-js.
  </action>

    Migrate each mutation's mutationFn:

    1. useCreatePropertyMutation — mutationFn (data: PropertyCreate):
       - const supabase = createClient()
       - const { data: created, error } = await supabase.from('properties').insert({ ...data }).select().single()
       - if (error) handlePostgrestError(error, 'properties')

       - const supabase = createClient()
       - const updatePayload = version ? { ...data, version } : { ...data }
       - const { data: updated, error } = await supabase.from('properties').update(updatePayload).eq('id', id).select().single()
       - if (error) handlePostgrestError(error, 'properties')
       - return updated as Property
