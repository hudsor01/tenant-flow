  <action>
    Create `supabase/functions/export-user-data/index.ts` following the exact pattern from `send-tenant-invitation/index.ts`:

    1. CORS handling: `handleCorsOptions(req)` early return, `getCorsHeaders(req)` on all responses
    2. Method check: Only allow GET (not POST -- this is a download, no request body needed)
    3. Env validation: `validateEnv({ required: ['SUPABASE_URL', 'SUPABASE_SERVICE_ROLE_KEY', 'SUPABASE_ANON_KEY'] })`
    4. JWT auth guard: Extract Bearer token, create anon-key client, call `getUser(token)`, reject if invalid
    5. Service role client for data queries (bypasses RLS to get all user data)
    6. Determine user role: Query `users` table for `user_type` where `id = user.id`
    7. Role-aware data collection using `Promise.all` for independent queries:

    **OWNER data (user_type = 'OWNER'):**
    - `users` record (profile: full_name, email, first_name, last_name, phone, avatar_url, created_at, user_type, deletion_requested_at)
    - `properties` where `owner_user_id = user.id` (select id, name, address, city, state, zip, type, status, created_at)
    - `units` joined through properties (select id, unit_number, bedrooms, bathrooms, rent_amount, status, property_id)
    - `leases` where `owner_user_id = user.id` (select id, lease_status, start_date, end_date, rent_amount, created_at)
    - `tenants` joined through leases via lease_tenants (select id, first_name, last_name, email, phone from users where tenants.user_id)
    - `rent_due` joined through leases (select id, amount, due_date, status, lease_id)
    - `rent_payments` joined through leases (select id, amount, status, payment_date, lease_id)
    - `maintenance_requests` where `owner_user_id = user.id` (select id, title, description, status, priority, created_at)
    - `documents` where `owner_user_id = user.id` (select id, name, type, created_at)
    - `expenses` where `owner_user_id = user.id` (select id, description, amount, category, expense_date)

    **TENANT data (user_type = 'TENANT'):**
    - `users` record (same fields as owner)
    - `tenants` record where `user_id = user.id` (select id, status, date_of_birth, emergency_contact_name, emergency_contact_phone, emergency_contact_relationship)
    - `lease_tenants` joined to `leases` (select lease_id, responsibility_percentage, leases.start_date, leases.end_date, leases.rent_amount, leases.lease_status)
    - `rent_due` joined through lease_tenants -> leases (select id, amount, due_date, status)
    - `rent_payments` where the payment's lease is in the tenant's leases (select id, amount, status, payment_date)
    - `maintenance_requests` where `tenant_id` matches the tenant's tenant record id (select id, title, description, status, priority, created_at)

    8. Build response JSON:
    ```typescript
    const exportData = {
      exported_at: new Date().toISOString(),
      user_role: userType,
      user_id: user.id,
      // ... role-specific data sections
    }
    ```

    9. Return with download headers:
    ```typescript
    const filename = `tenantflow-data-export-${userType.toLowerCase()}-${new Date().toISOString().split('T')[0]}.json`
    return new Response(JSON.stringify(exportData, null, 2), {
      status: 200,
      headers: {
        ...getCorsHeaders(req),
        'Content-Type': 'application/json',
        'Content-Disposition': `attachment; filename="${filename}"`,
      },
    })
    ```

    10. Error handling: Wrap in try/catch, use `errorResponse()` for 500s

    IMPORTANT:
    - Use `Promise.all` (not `Promise.allSettled`) for parallel queries since any failure should abort
    - Use service_role client for all data queries (need to bypass RLS to get complete data)
    - Select specific columns (never `select('*')` on list queries per CLAUDE.md)
    - Apply `.limit(10000)` on all list queries as a safety bound
    - No rate limiting needed -- this is an authenticated endpoint (JWT required)

    For the Deno test file, follow the pattern from `send-tenant-invitation-test.ts`:
    - Test CORS preflight response
    - Test 405 for POST/PUT/DELETE methods
    - Test 401 without auth header
    - Test response headers (Content-Type, Content-Disposition)
    - Test response body structure (valid JSON, has expected_at, user_role keys)
    - Use `Deno.test()` with descriptive names
