<objective>
DATA-01: occupancy analytics render empty because `get_occupancy_trends_optimized` returns a jsonb ARRAY (`{month, occupancy_rate, total_units, occupied_units}` ordered month DESC), but every consumer treats it as an OBJECT — `fetchOccupancyTrends` falls back to `{}`, `occupancyPageData` reads `raw.metrics/raw.trends` off the array (all undefined), `occupancyMetrics` reads `result?.total_units` off the array (zeros), and `mapLeaseAnalytics` runs a `z.object` safeParse against the array (fails -> empty). Fix the fetch to return the typed array, add an array-aware occupancy mapper, and rewire the four consumers to derive occupancy data from the array. Per the locked scope note, `leasePageData`/`overviewPageData` are MIS-WIRED (they call the lease mapper on the occupancy RPC); make them derive OCCUPANCY data (the data the RPC actually provides) and leave lease-financial sub-shapes empty — do NOT invent a lease-analytics RPC. Capture the lease/overview mis-wire (and the tenant-report occupancy mis-wire) as follow-ups.

Purpose: The occupancy analytics + report views render real monthly trends and current-occupancy metrics from the array the RPC actually returns.
Output: Four files updated (fetch + mapper + two consumer files), typecheck green, mis-wires captured as follow-ups in the SUMMARY.
</objective>
