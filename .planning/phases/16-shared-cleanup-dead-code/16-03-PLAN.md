    ```json
    {
      "$schema": "https://unpkg.com/knip@5/schema.json",
      "entry": [
        "src/app/**/page.tsx",
        "src/app/**/layout.tsx",

        "src/app/**/not-found.tsx",

      ],
      "project": [
        "src/**/*.{ts,tsx}",
        "proxy.ts"
      ],
      "ignore": [
        "src/components/ui/tour.tsx",
        "src/test/**",
        "src/types/jest-dom.d.ts",
        "src/types/supabase.ts"
      ],
      "ignoreDependencies": [
        "@types/*"
      ]
    }
    ```

    Notes on ignore patterns:
    - `tour.tsx`: vendored Dice UI component, has many internal-only exports
    - `src/test/**`: test utilities are entry points for test files, not production code
    - `jest-dom.d.ts`: ambient type declaration, no explicit imports
    - `supabase.ts`: generated file, consumed via re-exports in other type files

    **Step 3: Run Knip in debug mode first**
    ```bash
    npx knip --debug 2>&1 | head -50
    ```
    Review what plugins Knip auto-detected (Next.js, Vitest, ESLint, Tailwind, etc.). Adjust knip.json entry points if the plugins already cover them (avoid duplicate entries).

    **Step 4: Run Knip and capture findings**
    ```bash
    npx knip 2>&1
    ```

    **Step 5: For EVERY finding, independently verify before action.**
    Knip is a discovery tool, NOT an authority (per user decision). For each finding:

    a. **Unused files:** `grep -r "filename" src/ supabase/ --include="*.ts" --include="*.tsx" -l` to confirm zero importers. Also check for dynamic `import()` patterns and config file references (next.config.ts, vitest.config.ts, eslint.config.js).

    b. **Unused exports:** `grep -r "exportName" src/ supabase/ --include="*.ts" --include="*.tsx" -l` to confirm zero consumers. Check if the export is used in tests, as a type (may not appear in runtime imports), or re-exported.

    c. **Unused dependencies:** For each dependency Knip flags:
       - Check if it is a peer dependency of another used package (`pnpm why <pkg>`)
       - Check if it is used in config files (postcss.config.js, tailwind.config, eslint.config.js, next.config.ts)
       - Check if it is a plugin loaded by name (e.g., Vitest plugins, Babel plugins)
       - Check if it is a `@types/*` package needed for ambient types
       - Only remove after confirming it is genuinely unused

    d. **Action on confirmed dead code:**
       - Unused files: delete with `git rm`
       - Unused exports: remove the export keyword (if function is also unused locally, delete the function)
       - Unused dependencies: `pnpm remove <pkg>`

    **Step 6: Re-run Knip to confirm zero findings**
    ```bash
    npx knip
    ```
    If new findings appear (removing dead code can cascade), repeat the verify-then-delete cycle.

    **Step 7: Run full validation**
    ```bash
    pnpm typecheck && pnpm lint && pnpm test:unit -- --run
    ```
    All three must pass. If any test fails due to removed exports, restore the export and re-evaluate.
