**Part A: stripe-autopay-charge Edge Function**

Modify `stripe-autopay-charge/index.ts`:

**PAY-14: Per-tenant portion** — The pg_cron function will pass the pre-computed tenant amount (after 02-03 migration). However, as a safety net, also verify the amount by querying `lease_tenants.responsibility_percentage` and confirming the passed amount matches `rent_due.amount * percentage / 100`. If mismatch, log warning but use the computed value (not the passed value).

**PAY-08: Idempotency key** — Add `idempotencyKey` to `stripe.paymentIntents.create()`: use `${rent_due_id}_${tenant_id}` as the key. This ensures Stripe deduplicates if the same charge is attempted twice for the same rent_due and tenant.

**PAY-17: Update Stripe client** — Initialize with `apiVersion: '2026-02-25.clover'`.

**PAY-13: Retry tracking + failure notifications (Decision #9)** — After a failed PaymentIntent creation:
- Call an RPC or direct update to increment `rent_due.autopay_attempts`, set `autopay_last_attempt_at = now()`, and compute `autopay_next_retry_at` based on attempt number:
  - Attempt 1 failed: next retry in 2 days (day 3)
  - Attempt 2 failed: next retry in 4 days (day 7)
  - Attempt 3 failed: no more retries, mark as final failure
- Use service_role supabase client for the update.

- **Email tenant on EVERY failed attempt** using `sendEmail` from `../_shared/resend.ts`:
  - Import: `import { sendEmail } from '../_shared/resend.ts'`
  - Look up tenant email: query `tenants` table joined to `users` via `user_id` to get email and first_name.
  - Subject: `Autopay Failed - Attempt ${attemptNumber} of 3`
  - Body must include:
    - Attempt number (e.g., "Attempt 2 of 3")
    - Amount that failed to charge (tenant's portion, formatted as dollars)
    - A "Pay Now" deep link: `${FRONTEND_URL}/tenant/payments?rent_due_id=${rent_due_id}`
    - If not the final attempt: "We'll retry automatically on [next_retry_date]."
  - Tags: `[{ name: 'category', value: 'autopay_failure' }, { name: 'attempt', value: String(attemptNumber) }]`

- **Email owner ONLY on final failure (attempt 3)**:
  - Look up the owner via the lease: `leases.owner_user_id` -> `users` table for email and first_name.
  - Subject: `Autopay Failed for [tenant name] - Manual Follow-up Required`
  - Body: tenant name, property/unit info, amount, all 3 attempts failed, suggest contacting tenant.
  - Tags: `[{ name: 'category', value: 'autopay_final_failure' }]`

- sendEmail never throws (returns `{ success, error }`), so log any email failures but do not fail the function.
