**PAY-07: Create detach-payment-method Edge Function** — Create `supabase/functions/detach-payment-method/index.ts`. Per Decision #20, Stripe API detach is mandatory. Do NOT add any DB-only fallback or escape hatch — the Stripe `paymentMethods.detach` call MUST happen before DB deletion.
- Accept POST with `{ payment_method_id }` (the UUID from our `payment_methods` table).
- Authenticate the caller: extract Bearer token, verify via `supabase.auth.getUser(token)`.
- Look up the `payment_methods` row to get `stripe_payment_method_id` and verify the method belongs to the caller's tenant (join `tenants` on `user_id`).
- Call `stripe.paymentMethods.detach(stripePaymentMethodId)` to detach from Stripe Customer.
- Delete the row from `payment_methods` table.
- If the deleted method was `is_default = true`, promote the next most recent method to default.
- If the deleted method was used for autopay and no other methods remain, disable autopay via `toggle_autopay` RPC.
- Return 200 on success, 400/403/500 on error.
- Use CORS helpers from `../_shared/cors.ts`.
- Initialize Stripe with `apiVersion: '2026-02-25.clover'`.

**PAY-07: useDeletePaymentMethod** — Update the hook:
1. Check if this is the last payment method (query count). If count === 1, throw an error or return early with a toast message: "Add another payment method before removing this one."
2. Call the Edge Function: `supabase.functions.invoke('detach-payment-method', { body: { payment_method_id } })`.
3. If the deleted method was the default and autopay was active, the Edge Function handles auto-promotion. Invalidate query keys on success.
4. Show toast notification via sonner on success.

**PAY-07 UI: tenant-payment-methods.client.tsx** — Update the UI:
- Hide or disable the delete button when `paymentMethods.length === 1`.
- Show helper text: "Add another payment method before removing this one" when only one method exists.
- Add warning toast when deleting a method that is currently used for autopay.

Ensure all mutations invalidate `paymentMethodsKeys.all` and related dashboard query keys.
