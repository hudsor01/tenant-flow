**Step 1: Investigate the sync mechanism** — Run diagnostic queries against the live database to understand the Stripe sync engine:

```sql
-- Check stripe.migrations for setup history
SELECT * FROM stripe.migrations ORDER BY created_at;

-- Check last synced data across key tables
SELECT 'subscriptions' as tbl, max(created) as last_record FROM stripe.subscriptions
UNION ALL SELECT 'invoices', max(created) FROM stripe.invoices
UNION ALL SELECT 'charges', max(created) FROM stripe.charges
UNION ALL SELECT 'customers', max(created) FROM stripe.customers;

-- Check webhook events for recent activity
SELECT event_type, count(*), max(created_at) as last_seen
FROM stripe.webhook_events
GROUP BY event_type
ORDER BY last_seen DESC
LIMIT 20;

-- Check for webhook failures
SELECT * FROM stripe.webhook_failures ORDER BY created_at DESC LIMIT 20;

-- Check the materialized views
SELECT * FROM stripe.webhook_health_summary;

-- Look for triggers or functions that populate stripe tables
SELECT trigger_name, event_manipulation, action_statement
FROM information_schema.triggers
WHERE trigger_schema = 'stripe';

-- Check if there's a Supabase Stripe extension
SELECT * FROM pg_extension WHERE extname LIKE '%stripe%';
```

**Step 2: Determine the sync mechanism** — Based on the investigation, the sync is likely one of:
a) **Supabase Stripe Extension** (`stripe_fdw` or similar) — check `pg_extension`
b) **Custom webhook handler** that inserts into stripe.* tables — check webhook handler code
c) **Stripe's official Postgres sync** via their data pipeline product

**Step 3: Fix based on findings** — Create a migration that addresses the root cause:
- If it's a webhook-driven sync: check if the webhook endpoint in Stripe Dashboard is still active and pointing to the correct URL. The fix may be a dashboard configuration (checkpoint task).
- If it's an extension: re-enable or update it.
- If it's a cron job or scheduled function: find and fix it.

**Step 4: Backfill missing data** — After fixing the sync mechanism:
- For webhook-driven sync: use Stripe CLI or API to replay events from 2025-12-11 to present: `stripe events resend --since=2025-12-11`
- For extension-based sync: the extension should handle backfill on reconnection.
- Document the approach in the migration file as comments.

Note: The migration file may be minimal SQL (e.g., re-enabling a trigger or extension) with most work being diagnostic. The executor should document findings as comments in the migration file for future reference.
