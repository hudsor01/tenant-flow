# Phase 42 — UI Design Contract

> Visual and interaction contract for the 3-state in-app cancel flow that replaces `subscription-cancel-section.tsx`'s broken Stripe Portal redirect. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

Scope: `src/components/settings/sections/subscription-cancel-section.tsx` transforms from a single-state raw-`<button>` section into a 3-state state machine (Active → Cancel-scheduled → Canceled) backed by the shadcn AlertDialog primitive and the new `useCancelSubscriptionMutation` / `useReactivateSubscriptionMutation` hooks. Related new component: `src/components/settings/gdpr-data-actions.tsx` (extracted reusable GDPR export + delete subcomponent, per RESEARCH.md recommendation).

## Spacing Scale

Declared values (all Tailwind multiples of 4, inherited from project globals):

| Token | Value | Usage in this phase |
|-------|-------|----------------------|

**Button groups:** `flex gap-3` horizontal, reverse-stack on mobile (`flex-col-reverse sm:flex-row`) to match `AlertDialogFooter` convention.

Exceptions: none. All spacing snaps to 4-pixel grid.

## Color

The section's visual identity is **state-driven** — the card's border + bg tint communicates the state at a glance. Matches the existing `SubscriptionStatusBanner` tricolor system.

| Role | Value (Tailwind) | Usage |
|------|------------------|-------|
| Dominant (60%) | `bg-background` / `bg-card` | Default section surface (canceled-state GDPR subsection) |
| Secondary (30%) | `border-border` / `text-muted-foreground` | Card borders, hint copy, inert backgrounds |
| Accent (10%) | `text-destructive` / `border-destructive/20` / `bg-destructive/5` | Active-state "Danger Zone" framing + primary destructive action |
| Destructive | `bg-destructive` / `text-destructive-foreground` (on filled buttons) | Primary confirmation button inside dialog |
| Warning (grace window) | `border-amber-500/30` / `bg-amber-500/5` / `text-amber-600|700` | Cancel-scheduled inline alert (matches `account-danger-section.tsx:158` exactly) |
| Locked/canceled (terminal) | `border-red-300` / `bg-red-50` / `text-red-800` + dark variants (`dark:border-red-700 dark:bg-red-950/50 dark:text-red-200`) | Canceled-state header banner (matches `SubscriptionStatusBanner` locked banner exactly) |

**Accent reserved for** (explicit list — never "all interactive elements"):
- Active-state section card border/bg tint (`border-destructive/20 bg-destructive/5`)
- "Cancel Plan" trigger button text/border in Active state (`text-destructive border-destructive/20 hover:bg-destructive/10`)
- `AlertDialogAction` confirmation button (`variant="destructive"` full-fill `bg-destructive text-destructive-foreground`)
- Trash2 icon inside canceled-state "Request account deletion" button (inherited from `gdpr-data-actions` subcomponent)

**Non-destructive actions** (Reactivate, Export, dialog Cancel) use `variant="outline"` — no accent color, neutral surface.

**Dark-mode:** Every tinted surface declares a `dark:` variant (matches existing patterns). No bare `bg-white` (CLAUDE.md Accessibility Rule).

## Visual States (layouts + interaction wiring)

All 3 states render from the same `SubscriptionCancelSection` component, branched on `subscriptionStatus` and `cancelAtPeriodEnd` from `useSubscriptionStatus()`.

### State 1 — Active (default paying subscription)

**Condition:** `subscriptionStatus === 'active' && cancelAtPeriodEnd === false`

**Layout** (visual description for executor):
```
┌─────────────────────────────────────────────────┐
│ [AlertTriangle-sm] DANGER ZONE                  │  ← eyebrow, destructive color
│                                                 │
│ Cancel Subscription                             │  ← text-sm font-medium (body heading)
│ Your data stays for 30 days after cancellation. │  ← text-xs muted caption
│                              [Cancel Plan  ▸]   │  ← Button variant=outline, destructive text/border
└─────────────────────────────────────────────────┘   rounded-lg border-destructive/20 bg-destructive/5 p-6
                                                      flex items-center justify-between (matches current)
```

**Card classes:** `rounded-lg border border-destructive/20 bg-destructive/5 p-6`
**Layout inside card:** `flex items-center justify-between` (preserves existing spatial logic)
**Trigger button:** `<AlertDialogTrigger asChild><Button variant="outline" className="text-destructive border-destructive/20 hover:bg-destructive/10">Cancel Plan</Button></AlertDialogTrigger>`

**Interaction contract:**
- Click "Cancel Plan" → opens AlertDialog (Radix handles focus trap + overlay + Escape)
- Inside dialog, click "Yes, cancel plan" → `useCancelSubscriptionMutation().mutate()` fires
- While pending: confirm button shows `<Loader2 className="animate-spin" />` + "Canceling..." label, disabled
- On success: dialog closes automatically (AlertDialogAction default), toast fires, state flips to 2 on next render after query invalidation
- On error: dialog stays open, toast fires, user can retry or dismiss

### State 2 — Cancel-scheduled (grace window)

**Condition:** `subscriptionStatus === 'active' && cancelAtPeriodEnd === true`

**Layout:**
```
┌─────────────────────────────────────────────────┐
│ [Clock-sm] SUBSCRIPTION                         │  ← eyebrow, muted color (NOT destructive)
│                                                 │
│ ┌─── inline alert (amber warning) ────────────┐ │
│ │ [Clock-md amber]  Subscription ends         │ │
│ │                   March 15, 2026            │ │  ← date bold
│ │                                              │ │
│ │                   Your plan will end on     │ │
│ │                   March 15, 2026            │ │
│ │                   (12 days remaining). You  │ │
│ │                   can keep your plan active │ │
│ │                   by reactivating before    │ │
│ │                   then.                     │ │
│ └──────────────────────────────────────────────┘ │
│                                                 │
│ [Undo2] Reactivate Plan                         │  ← Button variant=outline, primary color
│                                                 │
│ Your data stays for 30 days after your plan     │  ← text-xs muted caption
│ ends per our privacy policy.                    │
└─────────────────────────────────────────────────┘   rounded-lg border bg-card p-6 (neutral card)
```

**Card classes:** `rounded-lg border bg-card p-6` (neutral — not destructive, because the user is still a paying customer)
**Inline alert classes:** `flex items-start gap-3 rounded-lg border border-amber-500/30 bg-amber-500/5 p-4` (exact copy of `account-danger-section.tsx:158`)
**Reactivate button:** `<Button variant="outline"><Undo2 className="h-4 w-4" aria-hidden="true" /> Reactivate Plan</Button>`

**Interaction contract:**
- Click "Reactivate Plan" → `useReactivateSubscriptionMutation().mutate()` fires directly (NO confirmation dialog — reactivation is a non-destructive reverse)
- While pending: button shows Loader2 + "Reactivating..." label, disabled
- On success: toast fires, state flips to 1 on next render
- On error: toast fires, button re-enables for retry

**Condition:** `subscriptionStatus === 'canceled'`

**Layout:**

```
│ sooner.                                         │
│                                                 │
│ ─────────────────────────────────────────────   │  ← mt-8 (section divider via spacing, not <hr>)
│                                                 │
│ YOUR DATA                                       │  ← eyebrow muted
│ Under GDPR and CCPA, you have the right to      │  ← text-sm muted
│ download or delete your data before the 30-day  │
│ retention window ends.                          │
│                                                 │
│ [Download] Download My Data                     │  ← outline button
│ [Trash2  ] Request Account Deletion             │  ← outline destructive-colored button
│                                                 │
│ Need to come back? View plans →                 │  ← text-sm link, mt-6
└─────────────────────────────────────────────────┘   rounded-lg border-red-300 bg-red-50 ...
                                                      dark:border-red-700 dark:bg-red-950/50 p-6
```

**Card classes:** `rounded-lg border border-red-300 bg-red-50 p-6 dark:border-red-700 dark:bg-red-950/50`
**GDPR subsection:** mounted via `<GdprDataActions />` subcomponent — neutral background, stands out from red-tinted parent
**"View plans" link:** `<Link href="/pricing" className="text-sm text-muted-foreground underline underline-offset-4 hover:text-foreground">Need to come back? View plans</Link>`

**Interaction contract:**
- "Download My Data" — reuses exact mutation flow from `account-data-section.tsx:47-80` (Blob → anchor click → URL.revokeObjectURL)
- "Request Account Deletion" — opens inline type-to-confirm form (exact pattern from `account-danger-section.tsx:202-252` — "type DELETE" input)
- If deletion is already pending (user previously requested): GDPR section flips to show "Account scheduled for deletion — {days remaining}" warning box + "Cancel Deletion" button (matches `account-danger-section.tsx:156-186` exactly)

### Loading & error substates (all 3 states)

| Scenario | UI behavior |
|----------|-------------|
| `useSubscriptionStatus().isLoading` on first mount | Render skeleton: `<div className="h-32 rounded-lg border bg-muted animate-pulse" />` — no BlurFade |
| `useSubscriptionStatus().isError` | Render neutral fallback: `<section className="rounded-lg border bg-card p-6"><p className="text-sm text-muted-foreground">Subscription status unavailable. <button onClick={refetch} className="underline">Retry</button></p></section>` |
| Mutation pending | Button shows spinner + "…ing" label, `disabled={true}`, retains its variant color |
| Mutation error | Toast fires (sonner), button re-enables immediately, no inline error inside the section |
| `subscriptionStatus === null` (no sub at all) | Component returns `null` — the section is scoped to "cancel an existing subscription", not subscription acquisition. `SubscriptionStatusBanner` handles the "no sub" copy elsewhere. |
| `subscriptionStatus === 'past_due' | 'unpaid'` | Component returns `null` — billing issues surface via `SubscriptionStatusBanner` (existing system, not redesigned this phase) |

## Accessibility (a11y contract)

All requirements enforced by CLAUDE.md Accessibility Rules + this phase.

| Surface | Rule |
|---------|------|
| Every icon-only button | **Forbidden** in this phase — all buttons have text labels. Icons accompany text via `<Icon className="h-4 w-4" aria-hidden="true" />` |
| Every button with an icon+text | Icon gets `aria-hidden="true"`; button itself gets `aria-label` equal to the copy contract (e.g., "Cancel my subscription") |
| AlertDialog | Radix provides `role="alertdialog"`, `aria-labelledby={title}`, `aria-describedby={description}`, focus trap, Escape-to-dismiss automatically — **do not override** |
| AlertDialog open trigger | Initial focus goes to `AlertDialogCancel` (safe default — matches Radix and D2's "dismiss" default) |
| State transition announcement | On cancel success, fire `toast.success(...)` which sonner renders in an `aria-live="polite"` region (default sonner behavior — no extra wiring needed) |
| Destructive confirm button | Marked `type="button"` (never `type="submit"` — the dialog is not a form); disabled state keeps `aria-disabled="true"` during mutation |
| Color contrast | All red/amber/destructive text on tinted backgrounds verified against WCAG AA: `text-red-800` on `bg-red-50` ≥ 7:1, `text-destructive` on `bg-destructive/5` ≥ 4.5:1, `text-amber-700` on `bg-amber-500/5` ≥ 4.5:1 (these are the exact combos already used by `SubscriptionStatusBanner` + `account-danger-section`, so they inherit proven-good contrast) |
| Keyboard navigation | Tab order: Cancel Plan button → (dialog opens) → Keep my plan → Yes, cancel plan → Escape dismisses. State 2: Reactivate button only. State 3: Download → Delete → View plans link. Sequential and predictable. |
| Focus management | After dialog-initiated cancel mutation succeeds, focus returns to the section (Radix default restores focus to trigger — which is now removed because state flipped to 2; Radix falls back to document body. This is acceptable — section re-renders with new content). |
| Muted text color | `text-muted-foreground` everywhere — never bare `text-muted` |
| Background color | `bg-background` or `bg-card` — never bare `bg-white` (dark-mode safety) |
| Link distinguishability | "View plans" link in State 3 uses `underline underline-offset-4` — not relying on color alone |

**Screen reader flow** (scripted walk-through):
- State 1: "Danger Zone. Cancel Subscription. Your data stays for 30 days after cancellation… button Cancel my subscription."
- Click confirm → "Alert. Cancel your subscription? Your plan will stay active until March 15, 2026…" [focus lands on Keep my plan]
- Tab → "Confirm subscription cancellation, button."
- Press Enter → (after success) toast "Subscription set to cancel on March 15, 2026."
- State 2 re-renders: "Subscription. Subscription ends March 15, 2026. Your plan will end on March 15, 2026, 12 days remaining…"

## Defaults Applied (Auto Mode — flag for review)

Items where no explicit upstream decision existed; sensible defaults chosen per project conventions. Verify during checker pass:

1. **State 2 eyebrow icon = `Clock`** (not AlertTriangle) — rationale: scheduled cancel is informational, not urgent. AlertTriangle is reserved for Danger Zone (State 1 active) and in-grace-period deletion (existing `account-danger-section` pattern). Clock matches `account-danger-section.tsx:159` for grace-period semantics.
2. **State 3 eyebrow icon = `Lock`** — matches `SubscriptionStatusBanner` locked-state icon exactly. The section is the terminal "you are locked out" moment in the subscription lifecycle.
3. **Reactivate button variant = `outline`** (not `default`/primary) — rationale: Reactivate is not the loudest CTA on the page; the user is still on an active plan. Outline matches the "gentle nudge back" tone without competing visually with global nav CTAs.
4. **AlertDialog initial focus on Cancel (dismiss) button** — Radix default; matches the safety principle that destructive actions should require deliberate navigation to the destructive option.
5. **Days-remaining format = `{n} days remaining`** — matches `account-danger-section.tsx:168` formatting verbatim. Singular "1 day remaining" edge case not special-cased (consistent with existing precedent; deferred to executor if they notice `n === 1`).
6. **End-date format = `Month D, YYYY`** (e.g., "March 15, 2026") — matches `formatDeletionDate()` at `account-danger-section.tsx:14-18` (`toLocaleDateString('en-US', { year: 'numeric', month: 'long', day: 'numeric' })`). Helper will be extracted or duplicated into `subscription-cancel-section.tsx` (executor's call — RESEARCH.md Don't-Hand-Roll row suggests reuse).
7. **"View plans" link in State 3 is a text link, not a Button** — rationale: true cancellation is terminal; encouraging resubscription via a loud CTA feels dark-pattern. A subtle text link respects the user's choice while keeping the path available.
8. **BlurFade `delay={0.55}` preserved** from current component — matches sibling settings sections' stagger animation rhythm.

All 8 defaults are traceable to existing project patterns. None introduce new visual tokens.
