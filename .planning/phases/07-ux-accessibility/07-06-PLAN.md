**Part B: useUnsavedChangesWarning hook (UX-17)**

Create `src/hooks/use-unsaved-changes.ts`:

```typescript
'use client'

import { useEffect } from 'react'

/**
 * Warns user before navigating away from a page with unsaved form data.
 * Uses the browser's beforeunload event.
 *
 * @param isDirty - Whether the form has unsaved changes
 */
export function useUnsavedChangesWarning(isDirty: boolean) {
  useEffect(() => {
    if (!isDirty) return

    const handler = (e: BeforeUnloadEvent) => {
      e.preventDefault()
      // Modern browsers show their own generic message regardless of returnValue
    }

    window.addEventListener('beforeunload', handler)
    return () => window.removeEventListener('beforeunload', handler)
  }, [isDirty])
}
```

Then integrate into multi-step forms:

1. **Lease creation wizard** (`src/components/leases/create/lease-creation-wizard.tsx`):
   - Import `useUnsavedChangesWarning`
   - Determine dirty state from form step progress (e.g., `currentStep > 0` or TanStack Form's dirty state)
   - Call: `useUnsavedChangesWarning(currentStep > 0)`

2. **Property form** (`src/components/properties/property-form.tsx`):
   - Import `useUnsavedChangesWarning`
   - Use TanStack Form's `formApi.state.isDirty` or track manually
   - Call: `useUnsavedChangesWarning(isDirty)`

3. **Tenant invite form** (`src/components/tenants/tenant-invite-form.tsx`) -- if it exists, add the hook. If not, skip.

Check each form component for its state management approach and determine the correct dirty signal. The hook is simple -- the integration point varies.
