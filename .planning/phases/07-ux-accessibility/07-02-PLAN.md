<task type="auto">
  <name>Task 2: Add mobile sidebar keyboard accessibility with focus management</name>
  <files>src/components/shell/app-shell.tsx, src/components/shell/tenant-shell.tsx</files>
  <action>
For BOTH shells, add keyboard accessibility to the mobile sidebar overlay:

**1. Escape key handler (UX-20)**
Create a ref for the hamburger button trigger: `const triggerRef = useRef<HTMLButtonElement>(null)`.
Attach the ref to the hamburger button element.

Add a useEffect that listens for Escape key when sidebar is open:
```tsx
useEffect(() => {
  if (!sidebarOpen) return
  const handleKeyDown = (e: KeyboardEvent) => {
    if (e.key === 'Escape') {
      setSidebarOpen(false)
      triggerRef.current?.focus()
    }
  }
  window.addEventListener('keydown', handleKeyDown)
  return () => window.removeEventListener('keydown', handleKeyDown)
}, [sidebarOpen])
```

**2. Focus management**
When sidebar opens, focus the first focusable element inside the sidebar (or the close button).
When sidebar closes (via Escape or overlay click), return focus to the hamburger trigger button.

Update the overlay click handler to also return focus:
```tsx
onClick={() => {
  setSidebarOpen(false)
  triggerRef.current?.focus()
}}
```

**3. Focus trap within sidebar overlay**
Add `role="dialog"` and `aria-modal="true"` to the sidebar overlay container.
Implement a basic focus trap: on Tab at the last focusable element, wrap to the first. On Shift+Tab at the first, wrap to the last. This can be done in the same keydown handler.

NOTE: The sidebar overlay is a div, not a Radix Dialog. A full focus trap implementation involves querying focusable elements. Keep it pragmatic:
- Query `[role="dialog"] a, [role="dialog"] button, [role="dialog"] input`
- On Tab/Shift+Tab at boundaries, prevent default and focus the opposite end
- This is a progressive enhancement -- if complex, at minimum ensure Escape works and focus returns to trigger
