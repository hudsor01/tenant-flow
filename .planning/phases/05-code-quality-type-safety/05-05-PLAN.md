**Tour component verification (CODE-12):**

1. Fetch the Dice UI Tour component source from the upstream repository (https://github.com/sadmann7/diceui). Check the `packages/ui/src/tour/` or similar path for the official Tour component source.

2. Compare the project's `src/components/ui/tour.tsx` (1,732 lines) against the upstream:
   - If the code substantially matches upstream (same structure, same hooks, same logic), then this is a vendored copy and should be kept as-is for upstream compatibility. Document this decision. The 300-line rule applies to OUR code, not vendored components.
   - If the code has significant local modifications, identify what's custom vs upstream. Consider splitting into subcomponents: `tour-overlay.tsx`, `tour-step.tsx`, `tour-navigation.tsx`, `tour-spotlight.tsx`, etc.
   - If upstream has a newer version with improvements, update to match.

3. For the 2 `eslint-disable react-hooks/exhaustive-deps` suppressions in tour.tsx with `useAsRef`: check if upstream uses this same pattern. If yes, they are legitimate (upstream convention). If no, fix them.

**'use client' audit (CODE-15):**

4. Find all page files with `'use client'`:
   ```bash
   grep -rl "'use client'" src/app/**/page.tsx
   ```

5. For each page file, determine if it can become a Server Component:
   - Does the page file itself use `useState`, `useEffect`, `useCallback`, `useMemo`, `useRef`, or any custom hooks? -> Must stay client
   - Does it use event handlers (`onClick`, `onChange`, etc.)? -> Must stay client
   - Does it only render client components without using hooks itself? -> Can remove `'use client'` (it's implicit)
   - Does it export `generateMetadata` or `generateStaticParams`? -> Must be Server Component

6. For pages that CAN become Server Components:
   - Extract interactive parts into client child components in a `components/` subdirectory
   - Remove `'use client'` from the page
   - Import client children normally (Next.js handles the boundary)

7. For pages that MUST stay client: leave `'use client'` in place, document why.

8. **Also audit non-page 'use client' files** beyond the 64 page files. Check if any component files have unnecessary `'use client'`. This is a large audit — focus on obvious cases:
   - Components that only render JSX with no hooks or events
   - Utility/type/constant files with unnecessary directive
   - Components that import client components but don't use hooks themselves

Note: This audit is best-effort. The goal is meaningful improvement, not perfection. Pages that clearly don't need the directive should be converted. Ambiguous cases can stay.
