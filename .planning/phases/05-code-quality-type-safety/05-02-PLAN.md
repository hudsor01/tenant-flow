2. **Create mapper functions**: For each distinct RPC return type, create a mapper function that:
   - Takes `unknown` as input (the RPC `data` return)
   - Casts to `Record<string, unknown>` (or array thereof) internally
   - Maps each field with explicit `Number()`, `String()`, `Boolean()` conversions
   - Returns the properly typed object
   - Place mapper functions in the same file near the top, or in existing mapper files (e.g., `query-keys/tenant-mappers.ts` already exists)

3. **Pattern for each file** (from the count: use-payments 5, use-reports 3 [handled in Plan 01], use-inspections 5, use-tenant-portal 3, use-lease 4, use-tenant 2, use-analytics 2, use-profile 2, use-billing 5, query-keys/maintenance-keys 1, query-keys/inspection-keys 4, query-keys/tenant-keys 1, query-keys/lease-keys 4):

   ```typescript
   // BEFORE:
   const stats = data as unknown as MaintenanceStats

   // AFTER:
   function mapMaintenanceStats(data: unknown): MaintenanceStats {
     const raw = data as Record<string, unknown>
     return {
       total: Number(raw.total ?? 0),
       open: Number(raw.open ?? 0),
       // ... all fields
     }
   }
   const stats = mapMaintenanceStats(data)
   ```

4. **Consolidate duplicate local types (CODE-05)**: While fixing each file, check if local type definitions duplicate types in `src/shared/types/`. Check TYPES.md first. If a shared type exists, import it. If local type is unique, leave it (but consider moving to shared types if used across files).

6. **Fix column references (CODE-09)**: In `tenantPortalQueries.payments()`, replace:
   - `amount_cents` -> `amount` (column stores dollars as numeric(10,2))
   - `paid_at` -> `paid_date`
   Verify against supabase.ts generated types for `rent_payments` table.

7. **Fix isSuccessfulPaymentStatus (CODE-10)**: Find this function and ensure it checks against the correct status values: `pending | processing | succeeded | failed | canceled`. The "successful" statuses should be `succeeded` (and possibly `processing` depending on usage context).

8. **Fix useLeaseList select purity (CODE-08)**: In `use-lease.ts`, the `select` function calls `queryClient.setQueryData` which is a side effect. Move this cache-priming logic to a `useEffect` that runs when `query.data` changes:
   ```typescript
   const query = useQuery(listQuery)
   useEffect(() => {
     if (query.data?.data) {
       for (const lease of query.data.data) {
         queryClient.setQueryData(leaseQueries.detail(lease.id).queryKey, lease)
       }
     }
   }, [query.data, queryClient])
   return query
   ```
