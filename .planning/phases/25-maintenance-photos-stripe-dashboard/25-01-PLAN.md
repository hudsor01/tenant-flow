Create a migration file that:

1. Creates `maintenance_request_photos` table:
   - `id uuid primary key default gen_random_uuid()`
   - `maintenance_request_id uuid not null references public.maintenance_requests(id) on delete cascade`
   - `storage_path text not null`
   - `file_name text not null`
   - `file_size integer`
   - `mime_type text not null default 'image/jpeg'` with CHECK constraint: `mime_type in ('image/jpeg', 'image/png', 'image/webp')`
   - `uploaded_by uuid references public.users(id) on delete set null`
   - `created_at timestamptz default now() not null`

2. Enable RLS: `alter table public.maintenance_request_photos enable row level security;`

3. Create RLS policies (one policy per operation per role):
   - `owner_select_maintenance_photos` FOR SELECT: owner can view photos for requests where `maintenance_requests.owner_user_id = (select auth.uid())`
   - `tenant_select_maintenance_photos` FOR SELECT: tenant can view photos for requests where `maintenance_requests.tenant_id` matches their tenant record via `tenants.user_id = (select auth.uid())`
   - `tenant_insert_maintenance_photos` FOR INSERT WITH CHECK: tenant can insert photos for their own requests (same tenant subquery)

4. Create index: `create index idx_maintenance_request_photos_request_id on public.maintenance_request_photos(maintenance_request_id);`

5. Create storage bucket:
   ```sql
   insert into storage.buckets (id, name, public)
   values ('maintenance-photos', 'maintenance-photos', true)
   on conflict (id) do nothing;
   ```

6. Create storage policies:
   - Authenticated users can upload (INSERT) to `maintenance-photos` bucket
   - Authenticated users can view (SELECT) from `maintenance-photos` bucket

Do NOT use PostgreSQL ENUMs -- use text with CHECK constraint for mime_type.
Do NOT create a `FOR ALL` policy -- separate policies per operation per role.
Wrap `auth.uid()` in `(select auth.uid())` for performance.

After creating the migration, run `pnpm db:types` to regenerate `src/types/supabase.ts`.
