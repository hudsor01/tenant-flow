From Supabase docs (integration test pattern):
```typescript
import { assert, assertEquals } from 'jsr:@std/assert@1'
import { createClient } from 'npm:@supabase/supabase-js@2'
import 'jsr:@std/dotenv/load'

const supabaseUrl = Deno.env.get('SUPABASE_URL') ?? ''
const supabaseKey = Deno.env.get('SUPABASE_PUBLISHABLE_KEY') ?? ''

Deno.test('function name', async () => {
  const client = createClient(supabaseUrl, supabaseKey, {
    auth: { autoRefreshToken: false, persistSession: false, detectSessionInUrl: false }
  })
  const { data, error } = await client.functions.invoke('function-name', { body: {} })
  assertEquals(error, null)
})
```

From CLAUDE.md Edge Function conventions:
- Auth: extract Bearer token, then supabase.auth.getUser(token)
- CORS: getCorsHeaders(req) + handleCorsOptions(req) for preflight
- Env validation: validateEnv() from _shared/env.ts
- Error responses: errorResponse() from _shared/errors.ts (generic messages)
- Rate limiting: rateLimit() from _shared/rate-limit.ts on unauthenticated functions

stripe-webhooks key behaviors:
- Verifies Stripe webhook signature
- Routes events to handler modules (payment_intent.succeeded, invoice.payment_failed, etc.)
- Idempotency via stripe_webhook_events table
- Uses service_role client for DB writes

stripe-rent-checkout key behaviors:
- Requires authenticated user (tenant)
- Validates rent_due_id, lease_id, tenant_id
- Creates Stripe checkout session with correct amount (dollars * 100 for Stripe)
- Returns checkout session URL
- Rate limited (unauthenticated endpoint variant for some paths)

4. Write tests (~10-15 total):

   - OPTIONS request returns CORS headers (preflight)
   - Request without stripe-signature header returns error
   - Request with invalid stripe-signature returns error (signature verification failure)

   **Event routing tests:**
   - POST with a mock webhook payload invokes the function (may return error due to invalid signature in test env -- that's expected; test that the function responds without crashing)
   - Test that unrecognized event types are handled gracefully (logged, not errored)

   **Error handling tests:**
   - Missing required env vars cause appropriate error response (function should fail-fast)
   - Malformed JSON body returns error

   **Integration notes:**
   - Stripe webhook signature verification will fail in test environment because we don't have the real webhook signing secret. Tests should verify the function: (a) accepts the request format, (b) returns appropriate error codes for auth failures, (c) doesn't crash on malformed input.
   - For actual webhook processing tests, the function needs a valid Stripe signature. These are effectively smoke tests that verify the function is deployed and responsive.
   - Use `client.functions.invoke('stripe-webhooks', { body: payload, headers: { 'stripe-signature': 'test' } })` pattern.
   - If running locally via `supabase functions serve`, tests can hit the local function instance.

5. Add a `.env` file or use `jsr:@std/dotenv/load` to load `SUPABASE_URL` and `SUPABASE_PUBLISHABLE_KEY` from the environment.
