   - `"isolatedModules": false` --> `"isolatedModules": true` (line 17)
   - `"noUnusedLocals": false` --> `"noUnusedLocals": true` (line 30)
   - `"noUnusedParameters": false` --> `"noUnusedParameters": true` (line 31)

2. Run `pnpm typecheck` to see all resulting errors.

3. Fix every error:
   - **Unused parameters in callbacks:** Prefix with underscore (e.g., `(item, index)` --> `(item, _index)`) or remove if not needed at all (e.g., `(item) => ...`). Choose removal over underscore when the parameter position does not matter.
   - **Unused local variables:** Remove the variable entirely. If it was a destructured import, remove just that import.
   - **Unused imports:** Remove the import statement or the specific named import.
   - **isolatedModules issues:** If any file uses `const enum` or `export =`, convert to regular `enum` or `export default`/named exports. This is unlikely given the codebase already uses ESM.
   - **checkJs issues:** If any JS files under `include` paths produce errors, fix them. Given the 5 JS files are outside `src/` and `scripts/`, impact should be zero. If `eslint.config.js` or `postcss.config.mjs` are somehow picked up, add `// @ts-nocheck` at the top only as a last resort.

4. Run `pnpm typecheck` again to confirm zero errors.
5. Run `pnpm lint` to confirm no lint regressions.

IMPORTANT: Do NOT modify test files (tests/**) -- they are excluded from tsconfig. Do NOT modify src/test/** -- also excluded. Only fix files within the tsconfig include scope.

Use underscore prefix (`_param`) for intentionally unused callback parameters that are required by position (e.g., Express `(_req, res)`, array `.map((_item, index) => index)`). Remove the parameter entirely when position does not matter.
