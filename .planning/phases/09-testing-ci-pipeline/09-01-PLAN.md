<task type="auto">
  <name>Task 1: Update CI workflows -- add next build, E2E smoke, and remove RLS path filter</name>
  <files>.github/workflows/ci-cd.yml, .github/workflows/rls-security-tests.yml</files>
  <action>
1. In `.github/workflows/ci-cd.yml`:
   - After the "Lint & Typecheck" step, add a new step:
     ```yaml
     - name: Build
       run: pnpm build
       env:
         SKIP_ENV_VALIDATION: 'true'
     ```
   - Add a second job `e2e-smoke` that runs ONLY on push to main (not PRs). This job:
     - Uses `if: github.event_name == 'push' && github.ref == 'refs/heads/main'`
     - Runs after `checks` job (`needs: [checks]`)
     - Checks out code, sets up pnpm, node, installs deps
     - Installs Playwright browsers: `npx playwright install --with-deps chromium`
     - Runs ONLY smoke tests: `npx playwright test --config tests/e2e/playwright.config.ts --project=smoke --project=public`
     - Has env vars for E2E credentials from secrets: E2E_OWNER_EMAIL, E2E_OWNER_PASSWORD, NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY
     - Timeout: 15 minutes
     - Uses `continue-on-error: true` so smoke failures don't block the merge (informational)
   - NOTE: Do NOT add tests or coverage to the main `checks` job. Tests stay local per user decision.

2. In `.github/workflows/rls-security-tests.yml`:
   - Remove the `paths:` filter entirely from the `pull_request:` trigger so RLS tests run on every PR to main.
   - Keep the `schedule:` cron and `workflow_dispatch:` triggers unchanged.
   - The updated trigger section should be:
     ```yaml
     on:
       pull_request:
         branches: [main]
       schedule:
         - cron: '0 9 * * 1'
       workflow_dispatch:
     ```
  </action>
  <verify>
    <automated>cat .github/workflows/ci-cd.yml | grep -c "pnpm build" && cat .github/workflows/ci-cd.yml | grep -c "SKIP_ENV_VALIDATION" && cat .github/workflows/ci-cd.yml | grep -c "e2e-smoke" && cat .github/workflows/rls-security-tests.yml | grep -c "paths:" | grep "0"</automated>
  </verify>
  <done>CI workflow has `pnpm build` step with SKIP_ENV_VALIDATION, E2E smoke job on push-to-main, and RLS workflow runs on all PRs (no paths filter).</done>
</task>
