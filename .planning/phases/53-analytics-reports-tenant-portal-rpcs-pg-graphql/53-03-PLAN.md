**maintenance-keys.ts: Fix tenantPortal() stub — critical two-step tenant_id resolution:**

The `maintenance_requests.tenant_id` column references `tenants.id` (the tenant record's primary key), NOT `auth.uid()` directly. Using `user.id` (auth.uid()) as the tenant_id filter will return empty results because `maintenance_requests.tenant_id` holds `tenants.id`.

Replace the TODO stub in `maintenanceQueries.tenantPortal()` with:
```typescript
const supabase = createClient()
const { data: { user } } = await supabase.auth.getUser()
if (!user) return { requests: [], total: 0, open: 0, inProgress: 0, completed: 0 }

// Step 1: Resolve tenants.id from auth.uid()
// maintenance_requests.tenant_id references tenants.id, not auth.uid()
const { data: tenantRecord, error: tenantError } = await supabase
  .from('tenants')
  .select('id')
  .eq('user_id', user.id)
  .single()

if (tenantError || !tenantRecord) {
  // Tenant record not found — user may not be a tenant yet
  return { requests: [], total: 0, open: 0, inProgress: 0, completed: 0 }
}

// Step 2: Filter maintenance_requests by tenants.id
const { data, error, count } = await supabase
  .from('maintenance_requests')
  .select('id, title, description, priority, status, created_at, updated_at, completed_at, unit_id', { count: 'exact' })
  .eq('tenant_id', tenantRecord.id)
  .order('created_at', { ascending: false })

if (error) handlePostgrestError(error, 'maintenance_requests')

const requests = (data ?? []) as MaintenanceRequest[]
const total = count ?? 0
const open = requests.filter(r => r.status === 'open' || r.status === 'assigned').length
const inProgress = requests.filter(r => r.status === 'in_progress' || r.status === 'needs_reassignment').length
const completed = requests.filter(r => r.status === 'completed').length

return { requests, total, open, inProgress, completed }
```

Note: RLS on maintenance_requests already supports tenant reads via the existing `maintenance_requests_select` policy:
`USING (property_owner_id = get_current_property_owner_id() OR tenant_id = get_current_tenant_id())`
The `get_current_tenant_id()` function returns `SELECT id FROM tenants WHERE user_id = auth.uid()` — which is the same two-step resolution this query performs. No additional RLS migration is needed; the query will return results correctly once the tenant record exists.
