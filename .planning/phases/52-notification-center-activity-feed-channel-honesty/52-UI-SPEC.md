# Phase 52 ‚Äî UI Design Contract

> Visual and interaction contract for the Notification Center, Activity Feed & Channel Honesty phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This phase surfaces existing backend, it does not invent a new visual language.** Every contract below is derived from tokens already in `src/app/globals.css` and two established mirrors: `ExpiringLeasesWidget` (`src/components/dashboard/expiring-leases-widget.tsx`) for card/list density and `CommandDialog` (`src/components/ui/command.tsx`) for popover density (locked by D-02). The executor reuses primitives ‚Äî zero new npm deps, zero new shadcn registry blocks.

### Discretion resolved (auto_mode ‚Äî no user prompt)

| Open item (from D-discretion) | Decision | Rationale |
|-------------------------------|----------|-----------|
| Popover vs DropdownMenu for the bell | **Popover** (`#components/ui/popover`) | D-02 wants a scrollable 10-item list with a sticky header (Mark all read) and sticky footer (View all). Popover gives free layout; DropdownMenu's roving-focus item semantics fight a scroll region + non-item footer link. RESEARCH.md concurs (Alternatives table). |
| Activity card placement in the grid | Pair it with `ExpiringLeasesWidget` in a **2-col grid at the bottom of `/dashboard`** (`grid grid-cols-1 lg:grid-cols-2 gap-6`), replacing the current single-widget full-width `div` at `page.tsx:157`. | D-07 says "alongside Quick Actions / expiring leases." Expiring-leases currently sits alone full-width; pairing it with the activity card is the lowest-disruption way to honor D-07 and keeps both attention (expiring) and audit (activity) side by side. |
| Activity entry copy format | `{actor/verb} ¬∑ {entity}` primary line + relative timestamp secondary line, mirroring the expiring-leases row (title `text-sm`, meta `text-xs`). | Reuses the proven widget row; keeps audit trail scannable. |
| Empty states | `Empty` compound component (CLAUDE.md mandate). Popover uses a **compact inline** empty (icon + one line, no dashed border) because the full `Empty` card is oversized for a 320px popover. | `Empty` default padding (`p-6 md:p-12`) is page-scale; the popover needs a tight variant. |
| Loading skeletons | `Skeleton` rows (`h-12 w-full`, 3‚Äì5 count) mirroring `ExpiringLeasesWidget`'s loading branch. | Established pattern. |
| Mark-all-read confirmation | **No confirmation dialog.** | GitHub/Linear reference model (D-02/D-10); marking read is low-stakes and non-destructive. A confirm would add friction the reference model deliberately omits. |

## Spacing Scale

Declared values (Tailwind base `--spacing: 0.25rem`; all multiples of 4):

| Token | Value | Usage in this phase |
|-------|-------|--------------------|
| xs | 4px (`gap-1`, `-top-1`/`-right-1`) | Badge overlay offset; icon-to-count gap |
| sm | 8px (`gap-2`, `p-2`) | Header control cluster; bell icon-button padding; popover row icon gap; unread dot size (`size-2`) |
| ‚Äî | 12px (`gap-3`, `p-3`, `py-3`) | List-row vertical padding (notification + activity rows); row internal icon-to-text gap |
| md | 16px (`p-4`, `gap-4`) | Popover header/footer inset; settings section inner spacing |
| lg | 24px (`p-6`, `gap-6`) | Card padding (`CardHeader`/`CardContent`); dashboard grid gap; `/notifications` page block gap |
| xl | 32px (`p-8` / `md:p-12` via Empty) | Empty-state page padding |

**Popover geometry:** width `w-80` (320px) desktop, clamped to `min(calc(100vw-1rem), 20rem)` at 375px; body scroll region `max-h-96` (384px) ‚Äî matches `CommandList`'s `max-h-75` density envelope (D-02). Header and footer are outside the scroll region (sticky), each `px-4 py-2` with a `Separator`.

**Exceptions:** none. The 44px mobile touch minimum on the bell button (`--touch-target-min: 2.75rem`) and the 8px unread dot are both multiples of 4.

## Typography

Constrained set for this phase's surfaces (popover, `/notifications` inbox, activity card, badge). Font family `--font-sans` throughout. Two weights carry all semantics: **400 regular** and **600 semibold**.

| Role | Size | Weight | Line height | Applied to |
|------|------|--------|-------------|-----------|
| Heading | 16px (`text-base`) | 600 | 1.35 (`--leading-snug`) | `CardTitle` on activity card + `/notifications` page title; popover header title "Notifications" |
| Row title ‚Äî read | 14px (`text-sm`) | 400 | 1.35 | Read notification title; activity entry primary line |
| Row title ‚Äî unread | 14px (`text-sm`) | 600 | 1.35 | **Unread** notification title (weight is one of three unread signals ‚Äî see Interaction Contracts) |
| Metadata / caption | 12px (`text-xs`) | 400 | 1.5 (`--leading-normal`) | Relative timestamp, message/subtitle, empty-state description, "View all" affordance sublabel |
| Count badge | 12px (`text-xs`) | 600 | 1 | Unread count on the bell ("9+" cap) |

Read‚Üîunread title weight (400‚Üî600) is the primary typographic distinction. Do not introduce a fourth size or a third weight for these surfaces.

## Color

Tokens are oklch CSS custom properties in `globals.css`; both light and dark values already defined. Never hard-code hex/oklch in components ‚Äî reference the token utilities.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-background` (`bg-background`) | `/notifications` page canvas; app shell behind the popover |
| Secondary (30%) | `--color-card` (`bg-card`) / `--color-popover` (`bg-popover`) | Activity card surface; popover surface; every notification/activity row sits on this |
| Accent (10%) | `--color-primary` (`bg-primary` fill / `text-primary-text` text) | See reserved-for list below |
| Destructive | `--color-destructive` (icon fill tint) / `--color-destructive-text` (glyph) | The "finalize failed" notification icon chip only |

**Accent (primary) reserved for ‚Äî explicit list, nothing else:**
1. Unread count badge fill on the bell (`bg-primary text-primary-foreground`).
2. The unread leading dot on each unread row (`bg-primary`, `size-2`).
3. Unread-row surface tint (`bg-primary/5`).
4. Text affordances: "Mark all read" and "View all notifications" (`text-primary-text hover:underline` ‚Äî mirrors the expiring-leases "View all leases ‚Üí" link).
5. Focus ring (`--color-ring`, already global via `*:focus-visible`).

Accent is NOT applied to read rows, row hover (hover uses `bg-accent` / `bg-muted/50`, the neutral surface, per the widget mirror), timestamps, or body copy.

## Copywriting Contract

Load-bearing note: notification **title strings are baked into DB triggers** (`create_notification` calls in the leases/maintenance triggers). The executor must use these exact strings in the SQL so DB copy and UI copy never diverge.

| Element | Copy |
|---------|------|
| Primary CTA (popover footer) | **View all notifications** ‚Üí routes to `/notifications` (D-01) |
| Secondary action (popover + page header) | **Mark all read** (disabled when unread count is 0) |
| Notification title ‚Äî tenant signed | **Tenant signed the lease** |
| Notification title ‚Äî lease fully executed | **Lease fully executed** |
| Notification title ‚Äî finalize failed | **Lease signing needs attention** |
| Notification title ‚Äî maintenance created | **New maintenance request** |
| Notification title ‚Äî maintenance status changed | **Maintenance status changed** |
| Popover empty (compact) | Icon `Bell` + **You're all caught up** / no body |
| `/notifications` empty (full `Empty`) | Title: **No notifications yet** ¬∑ Description: **When a lease is signed or a maintenance request comes in, you'll see it here.** |
| Activity card empty (`Empty`, compact) | Title: **No activity yet** ¬∑ Description: **Your leases, edits, and uploads show up here as you work.** |
| Notifications load error | **Couldn't load notifications.** + `Retry` action (button re-runs the query) |
| Activity load error | **Couldn't load activity.** + `Retry` action |
| Bell aria-label | **Notifications** when 0 unread; **Notifications, {n} unread** when unread > 0 (icon-only button a11y ‚Äî CLAUDE.md) |

**Destructive actions:** none in this phase. Mark-all-read is non-destructive and ships **without a confirmation dialog** (GitHub/Linear reference model). No `ConfirmDialog`/`AlertDialog` is introduced.

**Copy voice:** sentence case, no em-dashes in user-facing strings, no emojis (Lucide only). Relative timestamps ("2h ago", "3 days ago") via the existing `date-fns` formatters used across widgets.

## Surface Layouts

### 1. NotificationBell (header client island ‚Äî NOTIF-02)
- Replaces the `<Link href="/settings?tab=notifications">` bell at `app-shell-header.tsx:131`. Mounts as a sibling of `GlobalSyncIndicator` inside the right-side cluster (`gap-2`).
- Trigger: ghost icon button, `p-2 rounded-md hover:bg-muted`, `Bell` at `w-5 h-5 text-muted-foreground` (unchanged from current), 44px touch target on mobile.
- Badge overlay: `Badge` positioned `absolute -top-1 -right-1`, `bg-primary text-primary-foreground`, `text-xs font-semibold`, min `size-4` rounded-full; content `count > 9 ? "9+" : String(count)` (D-11). Hidden entirely when count is 0.
- Poll: unread count via `refetchInterval: 60_000`, HEAD `count: 'exact'` query (no rows transferred).

### 2. Notification popover (NOTIF-02/03, density = CommandDialog per D-02)
- `PopoverContent align="end" sideOffset={8}`, `w-80 p-0` (override default `p-4`), clamped on mobile.
- **Header** (sticky, `px-4 py-2`, `Separator` below): title "Notifications" (16px/600) left; "Mark all read" (`text-primary-text text-sm`, disabled at 0 unread) right.
- **Body** (`max-h-96 overflow-y-auto`): up to 10 `NotificationItem` rows, `divide-y divide-border`. Empty ‚Üí compact inline empty. Loading ‚Üí 4 `Skeleton` rows.
- **Footer** (sticky, `px-4 py-2`, `Separator` above): "View all notifications" (`text-primary-text text-sm font-medium`, full-width, routes to `/notifications`).

### 3. NotificationItem row (shared by popover + `/notifications`)
Mirrors the expiring-leases row exactly. `Link href={action_url}` (app-relative only ‚Äî never external; open-redirect guard per RESEARCH Security Domain), `flex items-center gap-3 px-4 py-3 hover:bg-accent transition-colors`:
- Leading: type-icon chip (`size-8 rounded-lg flex-center` with the semantic `icon-bg-*` utility) OR, for unread emphasis, a `size-2 bg-primary rounded-full` dot before the icon.
- Center (`min-w-0 flex-1`): title line (14px, **600 if unread / 400 if read**, `truncate`) + message/relative-time line (12px `text-muted-foreground`, `truncate`).
- Trailing: `ChevronRight size-4 text-muted-foreground` (row is a shortcut to the entity ‚Äî D-03).
- Click = mark that item read + navigate (D-03/D-10). Opening the popover never auto-clears (D-10).

### 4. `/notifications` page (NOTIF-01/03, full inbox)
- Lives at `app/(owner)/notifications/page.tsx`, inside the owner app shell (bg-background page). Page block: `Card` shell or plain list, `max-w-3xl` column, `p-6` / `gap-6`.
- Page header: `h1` "Notifications" (typography-h1) + "Mark all read" action right-aligned.
- List: paginated via `.range()` + `{ count: 'exact' }` (bounded ‚Äî Pitfall 5), `divide-y divide-border`, same `NotificationItem` rows. Read and unread both shown (audit-complete inbox); unread visually distinct.
- Empty ‚Üí full `Empty` compound. Error ‚Üí error copy + Retry.

### 5. DashboardActivityCard (ACT-01/02)
- New `Card` mirroring `ExpiringLeasesWidget` verbatim in structure. Placed in the bottom 2-col grid alongside expiring-leases (discretion above).
- `CardHeader`: `CardTitle` with a leading lucide icon (`Activity`, `size-4 text-muted-foreground`) "Recent activity"; `CardDescription` "Everything happening across your portfolio".
- `CardContent`: up to 10 activity rows (`divide-y divide-border`, `py-3`), each = type-icon chip + primary line (`{action} ¬∑ {entityName}`, 14px/400) + relative timestamp (12px muted). No unread concept (audit trail, not attention ‚Äî D-08).
- Empty ‚Üí compact `Empty` ("No activity yet"). Loading ‚Üí 3 `Skeleton` rows.
- **ACT-02 disambiguation (D-08 attention-vs-audit):** notifications carry the unread dot, count badge, mark-read affordances, and `ChevronRight` deep-link chevron; activity rows carry none of those ‚Äî no dot, no badge, no chevron, no read state. This visual asymmetry is the design pass that keeps the two surfaces from reading as duplicates.

### 6. NotificationSettings (HONEST-01/02) ‚Äî modification
- File `src/components/settings/notification-settings.tsx`. Remove the **SMS** channel block (lines ~116‚Äì131) and the **Push** channel block (lines ~133‚Äì148) entirely, plus their `MessageSquare` and (channel) `Bell` icon imports if now unused.
- Remove the **In-App** channel Switch (in-app is always-on per D-05) ‚Äî keep the row's intent gone, not a disabled toggle; the badge/inbox are the truthful in-app surface.
- Keep: **Email** channel toggle + the three category toggles (Maintenance / Lease Updates / General). "Enable All" (`handleEnableAllToggle`) must stop writing `sms`/`push`/`inApp` and write only `email` + categories.
- No new visual language ‚Äî the section keeps its existing `rounded-lg border bg-card p-6` shells and `Switch` primitives.

## Interaction & State Contracts

| State | Contract |
|-------|----------|
| Unread distinction (3 signals) | (1) leading `bg-primary size-2` dot, (2) row surface `bg-primary/5`, (3) title weight 600. Read row = none of these (transparent surface, weight 400, no dot). |
| Row click | Marks the item read (optimistic), then navigates to `action_url` via `next/link`. Invalidate `notificationKeys` + `ownerDashboardKeys.all`. |
| Mark all read | Header/page button; clears every unread; no confirm; disabled at 0 unread. |
| Open popover | Never auto-marks read (D-10, badge integrity). |
| Badge cap | `count > 9 ? "9+" : count`; badge hidden at 0. |
| Loading | `Skeleton` rows (popover 4, card 3) ‚Äî never a spinner blocking the surface. |
| Empty | `Empty` compound (compact in popover/card, full on `/notifications`). |
| Error | Inline copy + Retry button that re-runs the query; never a blank surface. |
| Focus / keyboard | Popover trap + Escape close (radix default); bell button focus-visible ring (global). Rows are `Link`s ‚Üí keyboard-focusable, Enter activates. |
| Mobile (375px) | Popover width clamped to viewport; bell 44px touch target; `/notifications` and activity rows full-width, truncation preserved. |
| Motion | Popover uses radix `data-[state]` fade/zoom (already in `PopoverContent`); respects `prefers-reduced-motion` (global rule). No custom animation added. |
