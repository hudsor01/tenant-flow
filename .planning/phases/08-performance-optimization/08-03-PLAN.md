**stripe-autopay-charge/index.ts:**

After body validation (line ~100), steps 3, 3b, rent_due fetch, step 4 (connected account), and step 6 (unit -> property_id) are ALL independent -- they only depend on fields from the request body which are already validated. Refactor these 5 sequential queries into a single Promise.all:

```typescript
// Parallel: all 5 lookups depend only on body fields (already validated)
const [
  existingPaymentResult,
  leaseTenantsResult,
  rentDueResult,
  connectedAccountResult,
  unitResult,
] = await Promise.all([
  // Step 3: Duplicate payment check
  supabase.from('rent_payments').select('id')
    .eq('rent_due_id', rent_due_id).eq('tenant_id', tenant_id)
    .in('status', ['succeeded', 'processing']).maybeSingle(),
  // Step 3b: Lease tenant responsibility percentage
  supabase.from('lease_tenants').select('responsibility_percentage')
    .eq('lease_id', lease_id).eq('tenant_id', tenant_id).maybeSingle(),
  // rent_due: amount, due_date, attempts
  supabase.from('rent_due').select('amount, due_date, autopay_attempts')
    .eq('id', rent_due_id).single(),
  // Step 4: Connected account
  supabase.from('stripe_connected_accounts')
    .select('stripe_account_id, default_platform_fee_percent, charges_enabled')
    .eq('user_id', owner_user_id).maybeSingle(),
  // Step 6: Unit -> property_id
  supabase.from('units').select('property_id')
    .eq('id', unit_id).maybeSingle(),
])
```

Then handle each result's error/null check individually (same error handling as current code, just after the parallel fetch). Keep the same error response messages and status codes. The rest of the function (fee calculation, PaymentIntent creation, insert) stays sequential since it depends on parallel results.

```typescript
// Parallel: tenant, rent_due, and duplicate check are independent
const [tenantResult, rentDueResult, duplicateResult] = await Promise.all([
  // Step 2: Resolve tenant
  supabase.from('tenants').select('id, stripe_customer_id')
    .eq('user_id', user.id).maybeSingle(),
```


Keep all existing error handling, response messages, and status codes unchanged. Use Promise.all (not Promise.allSettled) since any failure should abort the operation.
