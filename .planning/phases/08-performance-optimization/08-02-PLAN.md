Output: Loading animation components, 18 chart files refactored to use dynamic imports, blog page refactored with extracted markdown-content component.
</objective>

   The strategy: Each chart COMPONENT file already has `'use client'` and imports from 'recharts'. The parent pages that render these chart components should use `next/dynamic` to lazily load them. However, since many chart files are imported by page-level client components that compose multiple charts, the cleanest approach is:

   For files that are LEAF chart components (imported by page components):
   - `chart-area-interactive.tsx`, `revenue-overview-chart.tsx`, `maintenance-charts.tsx` (components/analytics), `property-charts.tsx` (components/analytics), `lease-charts.tsx` (components/analytics)

   For each page-level component that imports a chart component, change to dynamic import:
   ```typescript
   import dynamic from 'next/dynamic'
   import { ChartLoadingSkeleton } from '#components/shared/chart-loading-skeleton'

   const ChartComponent = dynamic(
     () => import('./path-to-chart-component'),
     { ssr: false, loading: () => <ChartLoadingSkeleton /> }
   )
   ```

   IMPORTANT: If the chart-importing file IS the chart component itself (it directly imports from 'recharts' and is used by a page), then the CONSUMER of that component needs the dynamic import.    - Dashboard page: dynamically import `chart-area-interactive` and `revenue-overview-chart`
   - Report sections: dynamically import the chart within each section component
   - The `components/analytics/` shared chart files: dynamically import them where they're consumed

   For each of the 4 report sections (`financial-report-section.tsx`, `maintenance-report-section.tsx`, `property-report-section.tsx`, `tenant-report-section.tsx`): these directly import from 'recharts'. Wrap the chart portion in a dynamically imported sub-component or move the recharts-using part to a child component and dynamically import it. Use `<ChartLoadingSkeleton />` as the loading fallback.
