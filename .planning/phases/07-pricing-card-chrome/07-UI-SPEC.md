## Spacing Scale

The project's `--spacing-*` scale is a 4px-base ladder (`globals.css:189-228`). This phase
uses only existing rungs. The table below declares ONLY the rungs Phase 7 references in the
classes it authors or edits — it is not a full inventory of the pricing surface.

| Token | Value | globals.css var | Usage in Phase 7 |
|-------|-------|-----------------|------------------|
| xs | 4px | `--spacing-1` | Icon-to-text gaps inside badges (`gap-1`) |
| sm | 8px | `--spacing-2` | Per-card savings line top margin (`mt-2`); price-row flex gap (`gap-2` featured) |
| md | 16px | `--spacing-4` | Badge horizontal padding step on classes Phase 7 leaves intact (`px-4`) |

**Phase 7 spacing exceptions (introduced by this phase):**

| Exception | Value | Why it is not a `--spacing-*` rung | Status |
|-----------|-------|-------------------------------------|--------|
| `-translate-y-1/2` on the "Most Popular" badge wrapper | half the badge's own height (~14px) | Transform offset relative to the badge's intrinsic height, not a fixed px value — it is the locked CONS-05 fix (`top-0 -translate-y-1/2` replaces `-top-4`) | Introduced by Phase 7 — intentional, locked |

No other spacing exceptions. Pre-existing card padding, badge padding (`py-1.5`), and
feature-row check-icon offsets (`mt-0.5`) are NOT modified by this phase and are therefore
out of scope for this contract — they live unchanged in the existing components.

## Typography

Phase 7 introduces no new type sizes. It consumes the existing Tailwind/`globals.css` type
ramp on the pricing cards. The table below declares ONLY the size(s) and weight Phase 7
actually authors — the per-card "Save $X/year" savings line. All other card type sizes
(price `text-5xl`/`text-3xl`, titles `text-2xl`/`text-xl`, body/descriptions
`text-base`/`text-sm`, etc.) are pre-existing and unchanged — see `src/app/globals.css`
for the full `--text-*` ramp.

| Role | Size | Weight | Line Height | globals.css / Tailwind token |
|------|------|--------|-------------|------------------------------|
| Per-card savings line ("Save $X/year") | 12px (standard cards) / 14px (featured card) | 600 (semibold) | 1.5 body default | `text-xs` / `text-sm` |

**Size:** 1 size role authored by this phase, rendered as `text-xs` on the Starter/Max
standard cards and `text-sm` on the Growth featured card to match each card's existing
type scale. No other type sizes are added or changed.

**Weight:** Phase 7 introduces exactly 1 weight — `font-semibold` (600) on the savings line,
so the dollar figure reads as a benefit, not as body copy (CONS-10). The pre-existing card
weights (`font-normal` body copy, `font-bold` prices/titles) are unchanged and not part of
this contract.

**CONS-09 constraint:** the price + `/mo` suffix must never wrap. Achieved by
`whitespace-nowrap` on the price-row flex container in BOTH cards — no type-size change.
The `NumberFlow`/`formatCurrency` price element and the `text-sm`/`text-xs` `/mo` span
stay on one baseline-aligned line at every breakpoint, including mid-animation width
changes when the billing toggle flips.

## Color

The project palette is oklch `--color-*` tokens (`globals.css:159-180`). 60/30/10 split for
the pricing surface:

| Role | Value | globals.css var | Usage |
|------|-------|-----------------|-------|
| Dominant (60%) | `bg-background` | page surface oklch | Pricing page background, section surface |
| Secondary (30%) | `bg-card` / `bg-card/95` | card surface oklch | Standard + featured card bodies, comparison table |
| Accent (10%) | `--color-primary` oklch(0.54 0.23 257) (+ `--color-accent` oklch(0.6 0.17 160) in the featured card's animated gradient ring) | `--color-primary`, `--color-accent` | "Most Popular" badge fill (`bg-primary text-primary-foreground`), featured-card gradient ring (`from-primary via-accent to-primary`), primary CTA, feature-check icons on the featured card |
| Semantic — success | `--color-success` oklch(0.66 0.2 160) | `--color-success` | Per-card "Save $X/year" line (`text-success`); standard-card feature-check icons (`text-success`); already the project convention for the savings chip |
| Destructive | not used | `--color-destructive` | No destructive action in this phase |

Accent reserved for:
- The "Most Popular" badge fill (`bg-primary`) — the single pinned-tab decoration on the Growth card.
- The featured card's animated gradient ring (`from-primary via-accent to-primary`) — pre-existing, not changed.
- The primary CTA buttons ("Start 14-Day Free Trial", "Start Free").
- Feature-check icons on the featured card (`text-primary`).

**CONS-10 savings color:** the per-card savings line uses `text-success` (the green
`--color-success` token). This is the existing project convention — it was already the
color used by the old global savings badge. It signals a positive/money-saving outcome and
is distinct from the `--color-primary` accent so the savings figure does not compete with
the CTA. No new color is introduced.

**Hard constraint (cross-cutting v1.0 rule):** zero hex, zero `rgb()`, zero `bg-white`,
zero bare `text-muted`, zero inline `style`, zero inline `[NNNms]` durations. A PR that
introduces any of these fails the perfect-PR review gate. All Phase 7 classes are
token-backed Tailwind utilities (`bg-primary`, `text-success`, `bg-card`, etc.).

## Copywriting Contract

Phase 7 is a chrome/math phase. Copy is locked upstream:
- Tier descriptions are byte-frozen by Phase 4 (`pricing.ts` lines 100/133/167) — must NOT change.
- Tier numbers are frozen by Phase 5 (`$19`/`$49`/`$149` monthly; `$190`/`$490`/`$1,490` annual).

The only copy this phase authors is the per-card savings line. Wording is locked by
CONTEXT.md (Option D):

| Element | Copy |
|---------|------|
| Primary CTA — Starter | "Start Free" (locked — Phase 4/CONS-06; not edited here) |
| Primary CTA — Growth | "Start 14-Day Free Trial" (locked; not edited here) |
| Primary CTA — Max | "Contact Sales" (locked — Phase 5/CONS-06 canonical sales label; not edited here) |
| Per-card savings line | `Save $38/year` (Starter), `Save $98/year` (Growth), `Save $298/year` (Max) — rendered ONLY when the billing toggle is on Annual; hidden on Monthly. Dollar figure is computed (`plan.price.monthly × 2`, math-equivalent to `calculateAnnualSavings`) — never hard-coded — and passed through `formatCurrency` so any future 4-digit savings renders with a thousands separator. |
| Annual-total clarifier | `Billed annually ($190/year)` / `Billed annually ($490/year)` / `Billed annually ($1,490/year)` on Annual; `Billed monthly` on Monthly — pre-existing pattern, retained for arithmetic honesty alongside the rounded `/mo` figure. |
| Empty state | Not applicable — pricing cards always render the three tiers from static `pricing.ts` config; there is no data-fetch / zero-data path. |
| Error state | Not applicable to chrome — the only failure surface is Stripe checkout (`toast.error(...)` on `createCheckoutSession` failure), which is pre-existing and out of Phase 7 scope. |
| Destructive confirmation | Not applicable — no destructive action in this phase. |

**Removed copy (CONS-10):** the global "Save $X" badge in the billing-toggle bar of
`bento-pricing-section.tsx` is deleted. It showed only Growth's `$98` while positioned as
an all-plans figure — misleading. The per-card savings line replaces it with per-tier,
auditable numbers.

## Interaction Contract

| Surface | State | Behavior |
|---------|-------|----------|
| Billing toggle (shadcn `Switch`, `id="billing-toggle"`) | Monthly (default) | Cards show monthly price + `Billed monthly`; per-card savings line hidden |
| Billing toggle | Annual (`checked`) | Cards show monthly-equivalent price + `Billed annually ($X/year)`; per-card `Save $X/year` line visible in `text-success`; featured card also shows the struck-through monthly price |
| "Most Popular" badge | static | Centered on the Growth card's top edge (`absolute top-0 left-1/2 -translate-x-1/2 -translate-y-1/2 z-10`); fully contained, no overlap of the 2px gradient ring at any breakpoint (375px → 1440px); never clipped by the section's `overflow-hidden` (lift is ~14px, well within `section-spacing` clearance) |
| Price row (`$XX` + `/mo`) | all breakpoints, incl. mid-toggle animation | `whitespace-nowrap` on the flex container keeps price and suffix on one baseline-aligned line; never wraps even while `NumberFlow`/`formatCurrency` width animates |
| "+N more features" disclosure (standard card) | toggle | Pre-existing; unchanged by Phase 7 |

No new animation tokens. The badge transform and toggle transitions use existing
`transition-colors` / `duration-300` utilities already on the components.
