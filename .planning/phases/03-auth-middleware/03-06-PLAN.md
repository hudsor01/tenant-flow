<!-- Supabase auth email hook contract (Deno Edge Function) -->
Supabase Auth Hooks: When configured, Supabase sends a POST to the Edge Function
with the email payload instead of sending it directly. The hook receives:
```typescript
// Request body from Supabase auth hook
interface AuthEmailHookPayload {
  user: { email: string; user_metadata: Record<string, unknown> }
  email_data: {
    token: string           // confirmation/reset token
    token_hash: string      // hashed version for URL
    redirect_to: string     // where to redirect after
    email_action_type: 'signup' | 'recovery' | 'invite' | 'magiclink' | 'email_change'
  }
}
```

  <name>Task 2: Create auth-email-send Edge Function</name>
  <files>supabase/functions/auth-email-send/index.ts</files>
  <action>
    Create `supabase/functions/auth-email-send/index.ts` as a Supabase auth email hook handler.

    The Edge Function:
    1. Handle CORS preflight with `handleCorsOptions(req)`
    2. Accept POST requests only
    3. Verify the request comes from Supabase by checking the `Authorization` header contains the `SUPABASE_AUTH_HOOK_SECRET` env var (or the service role key — check Supabase docs for auth hook verification)
    4. Parse the request body as `AuthEmailHookPayload`
    5. Based on `email_data.email_action_type`, select the appropriate template:
       - `'signup'` -> `signupConfirmationEmail`
       - `'recovery'` -> `passwordResetEmail`
       - `'invite'` -> `invitationEmail`
       - `'magiclink'` -> `magicLinkEmail`
       - `'email_change'` -> `emailChangeEmail`
    6. Build the action URL using `NEXT_PUBLIC_APP_URL` + `/auth/callback?token_hash={token_hash}&type={type}` (or the appropriate callback path)
    7. Call `sendEmail` from `../_shared/resend.ts` with the template output
    8. Return 200 on success, 500 on failure (Supabase retries on non-2xx)
    9. Add Resend tags for tracking: `{ name: 'category', value: 'auth' }`, `{ name: 'type', value: email_action_type }`

    Important: This Edge Function must be registered as a Supabase Auth Hook in the Supabase Dashboard (Auth > Hooks > Send Email). Add a comment at the top of the file explaining this requirement.

    Follow existing Edge Function patterns:
    - Use `Deno.serve(async (req) => { ... })`
    - Import from relative paths (`../_shared/resend.ts`, `../_shared/cors.ts`)
    - Never throw — return error responses
  </action>
