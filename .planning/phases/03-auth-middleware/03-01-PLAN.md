  <action>
    Create `middleware.ts` at project root (NOT in src/ — Next.js requires it at root or src/ root):
    - Import `updateSession` from `@/lib/supabase/middleware`
    - Define PUBLIC_ROUTES array:
      ```
      '/', '/login', '/pricing', '/about', '/blog', '/contact', '/faq',
      '/features', '/help', '/privacy', '/terms', '/security-policy',
      '/support', '/resources', '/search', '/accept-invite',
      '/auth/callback', '/auth/confirm-email', '/auth/post-checkout',
      '/auth/update-password', '/auth/signout', '/auth/select-role'
      ```
    - Public route check: `pathname === route || pathname.startsWith(route + '/')` — if public, return `supabaseResponse` directly (token refresh still happens)
    - If `!user`: redirect to `/login` with `redirect` search param set to `pathname`
    - Role enforcement using `user.app_metadata?.user_type`:
      - `'TENANT'` on `/dashboard*` -> redirect `/tenant`
      - `'OWNER'` or `'ADMIN'` on `/tenant*` -> redirect `/dashboard`
      - `'PENDING'` or no user_type on non-`/auth/*` route -> redirect `/auth/select-role`
    - CRITICAL: When creating redirect responses, copy all cookies from `supabaseResponse` to the redirect response to prevent session loss (Pitfall 2 from research):
      ```typescript
      const redirectResponse = NextResponse.redirect(url)
      supabaseResponse.cookies.getAll().forEach(cookie => {
        redirectResponse.cookies.set(cookie.name, cookie.value, cookie)
      })
      return redirectResponse
      ```
    - Matcher config: `'/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$|monitoring|api/).*)'`
    - The `/api/` exclusion ensures Next.js API routes (like attach-payment-method) are not affected by middleware redirects
  </action>
