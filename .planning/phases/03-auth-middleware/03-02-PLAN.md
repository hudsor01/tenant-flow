<objective>
Harden session validation across the auth stack: replace getSession() with getUser(), unify query key systems, fix module-level client singleton, and update getCachedUser.

Purpose: getSession() reads JWT from cookies without server validation — an attacker can forge session data. getUser() validates with Supabase auth server. The dual query key system causes stale auth cache. Module-level client persists across requests.
Output: Hardened auth-provider, use-auth, get-cached-user, and confirm-email files, plus authKeys structure tests.
</objective>
