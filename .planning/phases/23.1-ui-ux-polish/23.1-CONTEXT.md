<decisions>
## Implementation Decisions

### Typography Consistency
- Standardize ALL page headings across the app to `typography-h1` — no more `text-2xl font-bold`, `text-2xl font-semibold`, `typography-h2`, or `typography-h3` for top-level page titles
- Owner pages affected: dashboard, properties, leases, tenants, units (maintenance already correct)
- Analytics pages affected: overview, financial, property-performance
- Tenant pages affected: payment-methods, autopay
- Display-level utilities (`typography-hero`, `typography-display-*`) already updated to use Playfair Display font

### Dark Mode Fixes
- pricing/success and pricing/cancel: Replace `bg-primary` full-page wrapper with `PageLayout` or `bg-background`
- pricing/complete: Replace `from-slate-50 to-white` gradient with `from-background to-background`

### Component Consistency
- Replace native `<select>` at analytics/overview/page.tsx:121 with shadcn Select
- Replace raw `<button>` elements across financials pages with shadcn Button (8+ instances)
- Replace raw `<input>` and `<select>` in tenant maintenance/new/page.tsx:129-155 with shadcn components
- Align Units page layout to `p-6 lg:p-8` pattern, use Skeleton loading, use Empty compound component

### Empty States
- Replace bare "No data" text in 6 chart components with Empty compound component (icon + title + description)
- Fix Payment History "Next Due" permanent Skeleton — wire to actual data or remove card
- Files: revenue-expense-chart.tsx, financial-charts.tsx, occupancy-charts.tsx, lease-charts.tsx, maintenance-charts.tsx, property-charts.tsx

### Credibility & Copy
- Create `src/config/social-proof.ts` with canonical numbers, reference everywhere
- Conflicting claims: "10,000+" / "35,000+" / "2,847" / "8,000+" users — consolidate to one number
- Conflicting ROI: "30-day" / "60-day" / "90-day" — pick one
- Fix tenant greeting: resolve from user metadata, fallback to "there" not "Tenant"

### Semantic Color Tokens
- Replace `text-amber-600 dark:text-amber-400` with `text-warning` in maintenance-view.client.tsx and tenant-stats-cards.tsx
- Replace `text-emerald-600 dark:text-emerald-400` and `text-red-600 dark:text-red-400` with semantic tokens where used outside the design system

### Claude's Discretion
- Exact social proof number to use (suggest "10,000+" as most commonly used)
- Chart empty state icon and description text per chart type
- Whether to keep or remove the Payment History "Next Due" card

</decisions>
