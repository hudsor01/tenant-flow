<objective>
Build the race-safe e-sign metering data layer: an append-only `esign_events` table, the atomic `meter_esign_send(owner, lease)` RPC that enforces the 25/calendar-month Growth cap via a per-owner transaction advisory lock (Max unbounded), and a param-less `get_esign_usage_current_month()` read RPC the Settings widget will call. This is METER-01's enforcement core; the edge-fn hook (Plan 02) and the Settings usage widget (Plan 05) consume these objects.

Purpose: METER-01 requires the cap enforced race-safely at the DB layer — the edge function is the only caller and holds tier context, but the check-then-insert must be atomic in one transaction (a bare `INSERT ... SELECT ... WHERE count < 25` is TOCTOU under READ COMMITTED). Mirrors the Phase 53 `lease_reminders` / `claim_lease_reminders` append-only + service_role-only discipline.
Output: One migration applied to prod (filename reconciled), regenerated `src/types/supabase.ts`, and the dual-client RLS integration test green.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated client → esign_events | owner may read only their own metered events; must never write directly (append-only) |
| edge function (service_role) → meter_esign_send | the only write path; end users must not call the RPC to forge/skip counting |
| concurrent same-owner sends → the 25 cap | two in-flight sends must not both consume the 25th slot |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-54-01 | Tampering (quota evasion) | concurrent sends exceeding 25 | mitigate | `pg_advisory_xact_lock('esign:'||owner)` serializes the check-then-insert; concurrency test asserts exactly one 25th insert |
| T-54-02 | Elevation / Tampering | client calling meter_esign_send directly or writing esign_events | mitigate | RPC revoked from public/anon/authenticated (service_role-only); esign_events has SELECT-only authenticated policy, no insert policy |
| T-54-03 | Information disclosure (IDOR) | one owner reading another owner's events | mitigate | `esign_events_select` scoped `owner_user_id = (select auth.uid())`; dual-client RLS test asserts isolation |
| T-54-04 | Elevation (search_path hijack) | SECURITY DEFINER function | mitigate | every function `set search_path = public`; no dynamic SQL |
| T-54-SC | Supply chain | npm/pip/cargo installs | accept | zero package installs this plan (zero-new-deps invariant) — nothing to audit |
</threat_model>
