<objective>
Hook the `meter_esign_send` RPC (Plan 01) into the `lease-signature` edge function on the `send` action only, immediately after the existing e-sign entitlement gate, so the 26th Growth send this month is refused with a 402 upgrade response before any lease state change; `resend` is left untouched (D-02). Purpose: METER-01 says the cap is enforced at the `lease-signature` send path after `checkTierEntitlement`; the DB RPC is the enforcement, this plan is the call site + the honest upgrade CTA. RESEARCH Pitfall 4: the send wrapper throws `new Error(error.error)` and loses the 402 status + upgrade_url, so without this the block would show a generic toast with no Upgrade action.
Output: the deployed `lease-signature` function metering sends, the Deno branch coverage proving send-meters / resend-doesn't, and the send mutation surfacing an Upgrade CTA on the over-cap 402.
</objective>
