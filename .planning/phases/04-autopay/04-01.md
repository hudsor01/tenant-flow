# 64-01: Schema + Autopay Edge Function

## Overview
Add the autopay charging infrastructure: a new Edge Function that creates off-session PaymentIntents, a pg_cron job that fires daily, a failure notification email template, and modifications to existing checkout/webhook handlers.

## Steps

### 1. Database Migration
**File:** `supabase/migrations/20260227160000_autopay_schema_and_cron.sql`

Schema changes:
- Add `autopay_payment_method_id text null` to `leases` table (stores the Stripe PM ID chosen for autopay)
- The `auto_pay_enabled` boolean already exists on leases (default false)
- The `stripe_customer_id` already exists on tenants
- Add index on `leases(auto_pay_enabled)` for the pg_cron query

pg_cron function `process_autopay_charges()`:
- SECURITY DEFINER function
- Queries: active leases with `auto_pay_enabled = true` joined to `rent_due` records where `due_date = current_date` and `status = 'pending'`
- For each match, calls `net.http_post()` to the `stripe-autopay-charge` Edge Function with payload: `{ tenant_id, lease_id, rent_due_id, amount, stripe_customer_id, autopay_payment_method_id, owner_user_id }`
- Skips if a `rent_payments` record already exists for this `rent_due_id` with status `succeeded` or `processing`
- pg_cron schedule: daily at 07:00 UTC (morning so tenants see charges during business hours)

### 2. Modify stripe-rent-checkout to save payment method
**File:** `supabase/functions/stripe-rent-checkout/index.ts`

Add `setup_future_usage: 'off_session'` to `payment_intent_data` in the Checkout Session creation. This tells Stripe to save the card for future off-session charges.

Also add `customer` and `customer_creation: 'if_required'` -- if tenant has `stripe_customer_id`, pass it; otherwise let Stripe create one and we save it in the webhook.

### 3. Create stripe-autopay-charge Edge Function
**File:** `supabase/functions/stripe-autopay-charge/index.ts`

This is a service-to-service function (called by pg_cron via pg_net, not by browsers). No CORS needed.

Auth: Accepts a `service_role_key` header for authentication (pg_net sends it).

Logic:
1. Parse request body: `{ tenant_id, lease_id, rent_due_id, amount, stripe_customer_id, autopay_payment_method_id, owner_user_id }`
2. Validate all fields present
3. Duplicate check: query `rent_payments` for existing `succeeded` payment on this `rent_due_id`
4. Resolve connected account from `stripe_connected_accounts` where `user_id = owner_user_id`
5. Verify `charges_enabled = true`
6. Calculate fees (same logic as stripe-rent-checkout)
7. Resolve property_id from unit via lease
8. Create PaymentIntent:
   ```ts
   stripe.paymentIntents.create({
     amount: amountCents,
     currency: 'usd',
     customer: stripe_customer_id,
     payment_method: autopay_payment_method_id,
     off_session: true,
     confirm: true,
     application_fee_amount: applicationFeeCents,
     transfer_data: { destination: connectedAccountId },
     metadata: { tenant_id, lease_id, property_id, unit_id, rent_due_id, amount, period_month, period_year, due_date, period_start, period_end, autopay: 'true' }
   })
   ```
9. Insert `rent_payments` record with status `processing` (webhook will update to `succeeded` or `failed`)
10. Return success/failure response

### 4. Update stripe-webhooks for autopay
**File:** `supabase/functions/stripe-webhooks/index.ts`

Changes:
- In `payment_intent.succeeded`: already handles creating/updating rent_payments -- no changes needed (the autopay charge flows through the same path)
- In `payment_intent.payment_failed`: add autopay failure email sending
  - Check metadata for `autopay: 'true'`
  - If autopay, send failure notification email to tenant
  - Include link to tenant portal for manual payment
- In `checkout.session.completed`: save `stripe_customer_id` to tenant record if not already set (from `setup_future_usage` flow)

### 5. Create autopay failure email template
**File:** `supabase/functions/stripe-webhooks/_templates/autopay-failed.tsx`

Template shows:
- "Autopay Payment Failed" heading
- Amount that failed to charge
- Property address
- Card last 4 digits
- Failure reason (from Stripe)
- CTA button: "Pay Manually" linking to `/tenant/payments`
- Note that Stripe will retry automatically

### 6. Update checkout.session.completed webhook handler
Save the customer ID back to the tenant record when a checkout session completes with `setup_future_usage`. Also save the payment method ID to the `payment_methods` table.

## Verification
- Migration applies cleanly
- Edge Function handles the destination charge pattern
- pg_cron function correctly identifies autopay-eligible records
- Webhook handler processes autopay payments identically to manual payments
- Failure email renders correctly
