Create the Edge Function at `supabase/functions/send-tenant-invitation/index.ts`. Follow the exact same structure as `auth-email-send/index.ts` and `tenant-invitation-accept/index.ts`.

**Request format:**
- POST with JSON body: `{ invitation_id: string }`
- Headers: `Authorization: Bearer {jwt}` (owner's access token)

**Edge Function logic:**
1. Handle CORS preflight with `handleCorsOptions(req)`.
2. Reject non-POST methods with 405.
3. Call `validateEnv({ required: ['SUPABASE_URL', 'SUPABASE_SERVICE_ROLE_KEY', 'SUPABASE_ANON_KEY', 'RESEND_API_KEY'], optional: ['NEXT_PUBLIC_APP_URL'] })`.
4. Extract and validate Bearer token from Authorization header. Use anon-key client + `getUser(token)` to verify JWT and get user identity (same pattern as tenant-invitation-accept).
5. Parse request body, validate `invitation_id` is present (400 if missing).
6. Use service role client to fetch the invitation: `tenant_invitations` joined with `users!tenant_invitations_owner_user_id_fkey(full_name)`, `properties(name)`, `units(unit_number)`. Select columns: `id, email, invitation_code, invitation_url, expires_at, status, owner_user_id`. Use `.eq('id', invitation_id).single()`.
7. If invitation not found, return 404 `{ error: 'Invitation not found' }`.
8. If `invitation.owner_user_id !== user.id`, return 403 `{ error: 'Not authorized' }`. This ensures only the owner who created the invitation can trigger the email.
9. If invitation status is 'accepted' or 'cancelled', return 400 `{ error: 'Invitation is no longer active' }`.
10. If invitation is expired (`expires_at < now`), return 410 `{ error: 'Invitation has expired' }`.
11. Resolve the joined fields (PostgREST may return arrays for joins -- handle like tenant-invitation-validate does with `Array.isArray` checks).
12. Build the accept URL: Use `invitation.invitation_url` if it exists, otherwise construct from `appUrl + '/accept-invite?code=' + invitation.invitation_code`. The `appUrl` comes from `env['NEXT_PUBLIC_APP_URL'] ?? Deno.env.get('NEXT_PUBLIC_APP_URL') ?? 'http://localhost:3050'`.
13. Call `tenantInvitationEmail({ acceptUrl, tenantEmail: invitation.email, ownerName, propertyName, unitNumber })`.
14. Call `sendEmail({ to: [invitation.email], subject: template.subject, html: template.html, tags: [{ name: 'category', value: 'tenant-invitation' }, { name: 'invitation_id', value: invitation_id }] })`.
15. If `sendEmail` returns `{ success: false }`, return 500 with generic error via `errorResponse()`.
16. Return 200 `{ sent: true, email_id: result.id }`.
17. Wrap everything in try/catch, use `errorResponse(req, 500, err, { action: 'send_tenant_invitation' })` for unexpected errors.

**Test file** at `supabase/functions/tests/send-tenant-invitation-test.ts`:
Follow the same pattern as `tenant-invitation-accept-test.ts`. Use `Deno.test()` with `functions.invoke()` or raw fetch against locally served functions. Tests should cover the behaviors listed above. Mark tests that require live Supabase as integration tests.

Do NOT add rate limiting to this function -- it requires authentication (JWT), so IP-based rate limiting is unnecessary. Only unauthenticated endpoints need rate limiting per CLAUDE.md conventions.
