### New UI Elements

#### 1. Invitation Table Row (inline in tenant table)

Renders pending/sent invitations as rows in the existing tenant table. Not a separate component file -- integrated into `tenants.tsx` data merging and rendered via an invitation-specific row variant.

| Column | Content | Styling |
|--------|---------|---------|
| Checkbox | Disabled (not selectable) | `opacity-50 pointer-events-none` on checkbox |
| Name | Email address (no registered name yet) | `text-foreground font-medium` -- same as tenant fullName styling |
| Email | Same email repeated or dash | `text-sm text-muted-foreground` |
| Phone | Dash | `text-sm text-muted-foreground` showing `--` |
| Property | Dash | `text-sm text-muted-foreground` showing `--` |
| Status | `<Badge variant="warning" size="default">Pending</Badge>` or `<Badge variant="info" size="default">Sent</Badge>` | Default badge size (12px text, `px-2 py-0.5`) |
| Lease | Dash | `text-sm text-muted-foreground` showing `--` |
| Actions | DropdownMenu (see below) | Three-dot `MoreHorizontal` trigger |

**Row styling:** Same as active tenant rows (`hover:bg-muted/50 transition-colors`). No muted rows, no border accents. Source: D-02 from CONTEXT.md.

**Relative timestamp:** Below the status badge or in the status column area, show "Sent X ago" using `formatDistanceToNow` from `date-fns` (already a project dependency). Font: `text-xs text-muted-foreground`. Source: D-03 from CONTEXT.md.

## Interaction States

### Invitation Row Actions

| Action | Trigger | Feedback | Duration |
|--------|---------|----------|----------|
| Resend | Click "Resend Invitation" in dropdown | `toast.success("Invitation resent to {email}")` | Immediate |
| Copy Link | Click "Copy Invitation Link" in dropdown | `navigator.clipboard.writeText(url)` then `toast.success("Invitation link copied to clipboard")` | Immediate |
| Cancel | Click "Cancel Invitation" in dropdown | Opens `ConfirmDialog`. On confirm: `toast.success("Invitation to {email} cancelled")`. Row removed from list via query invalidation. | Dialog is modal, blocks until confirmed or dismissed |

### Duplicate Detection (All 3 Consumers)

| Trigger | Feedback | Source |
|---------|----------|--------|
| `useCreateInvitation()` returns `{ status: 'duplicate', existing }` | `toast.info("{email} already has a pending invitation (sent X ago)")` with embedded "Resend" action button | D-07 |
| User clicks "Resend" button in toast | Calls `resend()` from `tenant-invite-mutation-options.ts`. Toast auto-dismisses. New `toast.success("Invitation resent to {email}")` appears. | D-07 |

### Accept-Invite Session Branching

| State | UI Rendered | Source |
|-------|-------------|--------|
| Not logged in | Signup form (existing) + "Already have an account? Log in to accept" link below | D-09 |
| Logged in | "Accept Invitation" button with user email context. No signup form shown. | D-08 |
| Logged in + accepting | Button disabled, text "Accepting..." | D-08 |
| Logged in + accepted | Existing `SuccessState` component (redirect to `/tenant` after 2s) | Existing |

## Accessibility Contract

| Element | Requirement |
|---------|-------------|
| Invitation row dropdown trigger | `aria-label="Actions for invitation to {email}"` |
| Disabled checkbox on invitation rows | `aria-disabled="true"` (via Radix Checkbox disabled prop) |
| Cancel confirmation dialog | Focus trapped inside dialog (handled by Radix AlertDialog) |
| "Log in to accept" link | Standard anchor semantics via Next.js `<Link>`, no additional ARIA needed |
| Status badges | Text content ("Pending", "Sent") provides accessible name -- no additional ARIA needed |
| Toast notifications | sonner handles `role="status"` and `aria-live="polite"` automatically |
| Copy to clipboard | Success toast provides non-visual confirmation of the clipboard action |
