## Per-Page Audit Protocol

For each page, execute this sequence:

1. **Navigate:** `mcp__claude-in-chrome__navigate` to `http://localhost:3050/{path}`
2. **Desktop (1440x900):**
   - `mcp__claude-in-chrome__resize_window` to 1440x900
   - `mcp__claude-in-chrome__read_page` -- verify layout, headings, key elements
   - `mcp__claude-in-chrome__read_console_messages` -- check for errors
   - Test primary interactions via `mcp__claude-in-chrome__javascript_tool`
3. **Tablet (768x1024):**
   - `mcp__claude-in-chrome__resize_window` to 768x1024
   - `mcp__claude-in-chrome__read_page` -- verify responsive layout changes
4. **Mobile (375x812):**
   - `mcp__claude-in-chrome__resize_window` to 375x812
   - `mcp__claude-in-chrome__read_page` -- verify mobile layout
   - Test mobile sidebar toggle (hamburger menu)
   - `mcp__claude-in-chrome__read_console_messages` -- check for mobile-specific errors
5. **Log result** in AUDIT-LOG table

## Fix-Inline Protocol

When an issue is found:
1. Document the issue in AUDIT-LOG with status `FIX`
2. Read the source component file to identify the fix
3. Apply the CSS/layout/spacing fix
4. Wait for Next.js Fast Refresh
5. Re-verify at the same viewport in Chrome MCP
6. Run `pnpm typecheck && pnpm lint` to confirm no regressions
7. Continue the audit
