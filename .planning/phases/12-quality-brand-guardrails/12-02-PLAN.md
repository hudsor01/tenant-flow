Build a minimal admin approve/reject surface in the existing `(admin)` route group so the owner
can publish or archive `in-review` drafts — replacing the manual Supabase-Studio status flip.

explicit Approve click. Mutations go through the existing `blogs_update_admin` RLS policy
(is_admin-gated, authenticated client — never service role on the frontend). On publish, the
public ISR pages are revalidated so the new post surfaces.

Approve/Reject actions, an admin-gated publish/reject server action that updates status and
revalidates, TanStack Query mutations, and tests for the gating + mutations.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser (admin client) → public.blogs | Authenticated update crosses RLS; only is_admin may write status |
| draft markdown → admin preview DOM | Model-generated markdown rendered in the admin browser |
| client → server action (publish/reject) | Status-change request crossing into the publish pipeline |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-05 | Elevation of Privilege | /admin/blog page + mutations | mitigate | Inherited (admin) layout is_admin wall (defense layer 2) + server-action re-checks is_admin (layer 3) + blogs_update_admin RLS (layer 4); a non-admin update is rejected at the DB even if it reaches the action |
| T-12-06 | Tampering | Frontend writing with elevated creds | mitigate | Only the authenticated cookie-aware client is used; service-role key is never imported into any frontend/server-action file in this plan |
| T-12-07 | Tampering (XSS) | Markdown preview of model-generated draft | mitigate | Render via existing MarkdownContent (rehype-raw + rehype-sanitize); never dangerouslySetInnerHTML |
| T-12-08 | Elevation of Privilege | revalidatePath abuse by non-admins | mitigate | revalidate only runs inside publishBlogPost AFTER the is_admin re-check and a successful RLS-gated update — a non-admin call fails before any revalidation; no public revalidate endpoint is added |
| T-12-09 | Repudiation/Integrity | Auto-publish bypassing human review | mitigate | No code path sets status='published' except publishBlogPost, which only runs on an explicit admin Approve click |
| T-12-SC | Tampering | npm/bun installs | accept | No new packages — reuses @tanstack/react-query, @supabase/ssr, react-markdown stack already installed; no install task |
</threat_model>
