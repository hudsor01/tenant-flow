**`maintenanceQueries.list(filters?)`:**
- Uses `supabase.from('maintenance_requests').select(MAINTENANCE_SELECT_COLUMNS, { count: 'exact' })`
- RLS automatically filters to `owner_user_id = auth.uid()` — no manual user_id filter needed
- Apply filters: `unit_id`, `property_id` (via unit join — skip if no join needed, keep simple), `priority`, `category` (if category column exists; check schema), `status`
- Note: the `maintenance_requests` table does NOT have a `property_id` column directly — filtering by property requires joining through units. If property_id filter is needed, join: `.select(MAINTENANCE_SELECT_COLUMNS + ', units!inner(property_id)', { count: 'exact' })` and filter `.eq('units.property_id', filters.property_id)`. Keep this optional — only join when property_id filter is present.
- Apply `limit`/`offset` range via `.range(offset, offset + limit - 1)`
- Return `PaginatedResponse<MaintenanceRequest>` shape matching property-keys.ts pattern
- Order by `created_at` descending

**`maintenanceQueries.detail(id)`:**
- `supabase.from('maintenance_requests').select(MAINTENANCE_SELECT_COLUMNS + ', vendors(id, name, trade)').eq('id', id).single()`
- Include vendor relation when fetching detail view

**`maintenanceQueries.stats()`:**
- Run parallel count queries for each status (open, assigned, in_progress, needs_reassignment, completed, cancelled, on_hold) using `{ count: 'exact', head: true }`
- Return a typed stats object: `{ open: number, assigned: number, in_progress: number, needs_reassignment: number, completed: number, cancelled: number, on_hold: number, total: number }`
- No longer calling `/api/v1/maintenance/stats` NestJS endpoint

**`maintenanceQueries.urgent()`:**
- Query where `priority IN ('high', 'urgent')` AND `status NOT IN ('completed', 'cancelled')`
- Use `.in('priority', ['high', 'urgent']).not('status', 'in', '("completed","cancelled")')`

**`maintenanceQueries.overdue()`:**
- Query where `scheduled_date < now()` AND `status NOT IN ('completed', 'cancelled')`
- Use `.lt('scheduled_date', new Date().toISOString()).not('status', 'in', '("completed","cancelled")')`

**`maintenanceQueries.tenantPortal()`:**
- This requires tenant-scoped filtering. The tenant portal shows requests by the currently-authenticated tenant user. Defer this with a `// TODO(phase-53): tenant portal maintenance queries require RPC with tenant context` comment. For now return an empty result with the same shape as before so the component doesn't crash: `{ requests: [], total: 0, open: 0, inProgress: 0, completed: 0 }`

All queryFns must use `createClient()` from `#lib/supabase/client` and `handlePostgrestError` from `#lib/postgrest-error-handler`.
