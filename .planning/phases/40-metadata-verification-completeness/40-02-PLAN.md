## createPageMetadata signature (from src/lib/seo/page-metadata.ts)

```typescript
interface PageMetadataConfig {
  title: string          // keyword phrase only — root template appends ' | TenantFlow'
  description: string    // 150-160 chars, keyword-rich
  path: string           // URL path, used for canonical + og:url
  noindex?: boolean      // optional robots directive (omit for Phase 40 — all pages indexed)
  ogImage?: string       // optional OG image override (omit — factory default is acceptable)
}

export function createPageMetadata(config: PageMetadataConfig): Metadata
```

Returns Next.js `Metadata` object with populated:
- `title`, `description`
- `alternates.canonical` (absolute URL via `getSiteUrl()`)
- `openGraph` (type, url, title, description, siteName, locale, images)
- `twitter` (card: summary_large_image, title, description, images)
- `robots` (only when `noindex: true`)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build-time Next.js -> runtime HTML | Metadata is static at build time. No runtime user input flows into `<title>` or `<meta>`. |
| search engine / social crawler -> rendered HTML | Crawlers read metadata. No auth surface. All 4 pages are public. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-02-01 | Information Disclosure | OG image, canonical URL | accept | `getSiteUrl()` reads `NEXT_PUBLIC_SITE_URL` env — public by design. Default OG image is a public marketing asset. No secrets leak. |
| T-40-02-02 | Denial of Service | Build-time metadata generation | accept | `createPageMetadata` is synchronous; no I/O. Zero runtime cost added. |
| T-40-02-03 | Spoofing (brand impersonation via duplicate title) | Root `title.template` + page title | mitigate | Per D-06: legal/support pages already lack inline `\| TenantFlow` suffix. Factory preserves titles as authored. Template `%s \| TenantFlow` appends brand exactly once. E2E spec from Plan 40-01 asserts absence of `\| TenantFlow \| TenantFlow` on all 7 paths. |
| T-40-02-04 | Tampering (ASVS V14 — canonical URL single source) | `alternates.canonical` on each page | mitigate | `createPageMetadata` derives canonical from `path` argument + `getSiteUrl()` — single source of truth. No per-page string concatenation drift. |

No NEW threats introduced. Plan 40-02 exercises proven tooling from Phase 33.
