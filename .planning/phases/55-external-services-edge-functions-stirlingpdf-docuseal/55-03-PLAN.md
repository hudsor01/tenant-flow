    Create supabase/functions/docuseal-webhook/index.ts. This is an inbound webhook handler — similar in structure to stripe-webhooks.

    Pattern: Follow stripe-webhooks exactly — public endpoint (no JWT required), payload validation, idempotent processing, return 500 on processing errors so DocuSeal retries.

    **Security / verification:**
    DocuSeal supports a shared secret sent as a header. Check for `DOCUSEAL_WEBHOOK_SECRET` env var:
    - If set: verify that the request header `x-docuseal-signature` (or `authorization`) matches the secret. If mismatch: return 400.
    - If not set: accept all requests (open endpoint). Log a console.warn that the endpoint is unauthenticated.
    - Per locked decision: follow the Stripe webhook pattern — open endpoint with payload verification.

    The DocuSeal webhook payload structure (from the NestJS docuseal-webhook.service.ts for reference):

    **form.completed event:**
    ```typescript
    {
      event_type: 'form.completed'  // or just check payload structure
      id: number                    // submitter ID
      submission_id: number
      role: string                  // 'Property Owner' or 'Tenant'
      email: string
      completed_at: string
      metadata?: { lease_id?: string }
    }
    ```

    **submission.completed event:**
    ```typescript
    {
      event_type: 'submission.completed'
      id: number                    // submission ID
      status: string
      documents?: Array<{ name: string; url: string }>
      completed_at?: string
      metadata?: { lease_id?: string }
    }
    ```

    **Idempotency:**
    For form.completed: Check if owner_signed_at (or tenant_signed_at) is already set before writing. If already set, return 200 immediately (idempotent — duplicate delivery).
    For submission.completed: Check if lease_status is already 'active' before processing. If already active, return 200 immediately.

    **Processing logic:**

    **form.completed handler:**
    1. Parse body as JSON, extract submission_id, role, email, completed_at, metadata.lease_id
    2. Find lease: first try `leases.docuseal_submission_id = String(submission_id)`, then fallback to `leases.id = metadata.lease_id`
    3. If lease not found: return 200 (not our document — ignore gracefully)
    4. Determine role: isOwner = role.toLowerCase().includes('owner'), isTenant = role.toLowerCase().includes('tenant')
    5. signedAt = completed_at || new Date().toISOString()
    6. If isOwner and !lease.owner_signed_at:
       - Update: `leases set owner_signed_at = signedAt, owner_signature_method = 'docuseal' where id = lease.id`
       - On error: return 500 so DocuSeal retries
    7. If isTenant and !lease.tenant_signed_at:
       - Update: `leases set tenant_signed_at = signedAt, tenant_signature_method = 'docuseal' where id = lease.id`
       - On error: return 500

    **submission.completed handler:**
    Triggered when ALL parties have signed. Atomically:
    1. Parse body, extract submission id, status, documents[0].url (signed document), metadata.lease_id
    2. Find lease by docuseal_submission_id or metadata.lease_id
    3. If not found: return 200
    4. Idempotency: if lease.lease_status === 'active': return 200 immediately
    5. Three atomic writes (sequential, on error any write fails → return 500 for DocuSeal retry):
       a. Update lease: `lease_status = 'active'` (flip from pending_signature → active per locked decision)
       b. Insert notification: `notifications` table with `{ user_id: lease.owner_user_id, title: 'Lease fully signed', message: 'Your lease has been signed by all parties and is now active.', notification_type: 'lease' }` — IMPORTANT: use 'lease' not 'lease_signed'; the notifications_notification_type_check constraint only allows ('maintenance', 'lease', 'payment', 'system')
       c. Optionally: Update `docuseal_document_url` on leases if that column exists (check schema — if not, skip)
    6. Return 200

    **Main handler:**
    ```typescript
    Deno.serve(async (req: Request) => {
      if (req.method === 'OPTIONS') return new Response('ok', { headers: corsHeaders })

      // Verify shared secret if configured
      const webhookSecret = Deno.env.get('DOCUSEAL_WEBHOOK_SECRET')
      if (webhookSecret) {
        const signature = req.headers.get('x-docuseal-signature') ?? req.headers.get('authorization')
        if (signature !== webhookSecret && signature !== `Bearer ${webhookSecret}`) {
          return new Response('Invalid signature', { status: 400 })
        }
      }

      const body = await req.json()

      // Determine event type — DocuSeal sends event_type field
      const eventType = body.event_type as string

      try {
        if (eventType === 'form.completed') {
          await handleFormCompleted(supabase, body)
        } else if (eventType === 'submission.completed') {
          await handleSubmissionCompleted(supabase, body)
        } else {
          console.log(`Unhandled DocuSeal event: ${eventType}`)
        }
        return new Response(JSON.stringify({ received: true }), {
          status: 200,
          headers: { ...corsHeaders, 'Content-Type': 'application/json' }
        })
      } catch (err) {
        return new Response(
          JSON.stringify({ error: err instanceof Error ? err.message : 'Processing failed' }),
          { status: 500, headers: { ...corsHeaders, 'Content-Type': 'application/json' } }
        )
      }
    })
    ```

    Use SUPABASE_SERVICE_ROLE_KEY client for all DB operations (same as stripe-webhooks).

    The notifications table structure (from stripe-webhooks pattern):
    `{ user_id: string, title: string, message: string, notification_type: string }`

    ```typescript
      'Access-Control-Allow-Headers': 'authorization, x-client-info, apikey, content-type, x-docuseal-signature',
    }
    ```
