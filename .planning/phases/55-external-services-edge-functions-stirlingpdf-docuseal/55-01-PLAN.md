  <action>
    Create a new Deno Edge Function at supabase/functions/generate-pdf/index.ts.

    Pattern: Follow stripe-webhooks structure (CORS headers, Deno.serve, env var reads at top).

    Authentication: JWT-authenticated endpoint (NOT public). Read and verify Bearer token from Authorization header using supabase.auth.getUser(token). Return 401 if missing or invalid.

    Environment variables:
    - STIRLING_PDF_URL: base URL of the self-hosted StirlingPDF k3s instance (e.g. https://pdf.thehudsonfam.com)
    - SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY: for auth verification

    Request format: Accept POST with JSON body:
    ```typescript
    {
      html: string        // HTML content to convert to PDF
      filename?: string   // suggested filename for Content-Disposition header (default: 'report.pdf')
    }
    ```

    StirlingPDF API call:
    - Use the /api/v1/convert/html/pdf endpoint on StirlingPDF (HTML-to-PDF conversion)
    - POST multipart/form-data with the HTML content as a field named "fileInput" (file upload) or use the JSON endpoint /api/v1/convert/html/pdf if it accepts JSON. Use /api/v1/misc/html-to-pdf which accepts JSON: { htmlContent: string }
    - Actually, check StirlingPDF docs. The cleanest approach is to POST the HTML as multipart form data to /api/v1/misc/html-to-pdf with field "htmlContent" as a plain text field.
    - Set AbortSignal.timeout(30_000) — 30-second timeout per locked decision
    - On timeout or fetch error: return 502 with JSON error (fail-fast, no retry — matching stripe-webhooks pattern)

    Response: On success, pipe the PDF bytes from StirlingPDF response directly to the client:
    - Content-Type: application/pdf
    - Content-Disposition: attachment; filename="{filename}"
    - Include CORS headers

    Error handling:
    - StirlingPDF returns non-2xx: return 502 with descriptive error message
    - Missing STIRLING_PDF_URL env var: return 500

    CORS headers object:
    ```typescript
    const corsHeaders = {
      'Access-Control-Allow-Origin': '*',
      'Access-Control-Allow-Headers': 'authorization, x-client-info, apikey, content-type',
    }
    ```

    Handle OPTIONS preflight (return 'ok' with cors headers).

    Full implementation should be ~70 lines. No external npm imports needed — only the standard supabase client from esm.sh.
