<objective>
Ship DT-01 (the assembled replacement shell) + DT-04 (faceted status filter) + DT-05 (column visibility) + DT-06 (always-on virtualization) + DT-07 (grid/table toggle) as a self-contained `PortfolioDataTable` component — built but NOT yet mounted. It consumes the Wave-1 outputs: `useClientDataTable` (05-01a) for state and `portfolioColumns` (05-01b) for the column/aria-sort/filter/label metadata. It composes the already-vendored `DataTableFacetedFilter`, `DataTableViewOptions`, and `DataTablePagination`, and virtualizes the tbody via `useVirtualizer` over the post-pagination page rows.

Purpose: this is the visual heart of the swap. The three risks it defuses (per 05-CONTEXT.md):
1. **Virtualization + sticky header + pagination** — combine an absolute-positioned virtualized `<tbody>` (`getTotalSize()` height) with a sticky `<thead>` and the pagination footer. Reuse the proven pattern from `leases-table.tsx` (a `ref`'d scroll container, `getVirtualItems().map`, `estimateSize`), but measure the portfolio row height to the UI-SPEC min (h-12 / 48px). Always-on (D-2): one code path, no threshold branch.
2. **Grid + mobile read the same nuqs state** (D-5) — when `viewMode === "grid"` render the card grid off the SAME `table.getRowModel()` rows (already filtered/sorted/paged by the table), not a parallel array. Filter/sort/page is the table's job regardless of view.
3. **State ownership** — the component is a pure consumer: it receives `data`, `viewMode`/`setViewMode`, and `columnVisibility`/`onColumnVisibilityChange` as props. It does NOT import `dashboard-store` or `dashboard-presets-store` (Plan 05-03 owns wiring those). This keeps the component testable in isolation and lets Plan 05-03 do the atomic swap without touching this file's internals.

Output:
- `src/components/dashboard/components/portfolio-data-table.tsx` — `PortfolioDataTable` (shell + virtualized tbody + grid/table switch).
- `src/components/dashboard/components/portfolio-data-table-toolbar.tsx` — `PortfolioDataTableToolbar` (search + faceted filter + view-options + grid/table toggle).
- `src/components/dashboard/components/__tests__/portfolio-data-table.test.tsx` — component tests.

NO mount in `dashboard.tsx` (Plan 05-03). NO presets store, NO nuqs page-wiring beyond what `useClientDataTable` already provides internally. NO deletion of the hand-rolled files (Plan 05-03).
</objective>
