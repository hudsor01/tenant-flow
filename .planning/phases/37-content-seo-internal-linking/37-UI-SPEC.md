## Spacing Scale

Declared values (multiples of 4, using Tailwind spacing tokens):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (gap-1) | Icon-to-text gaps (ArrowRight icon offset) |
| sm | 8px (gap-2) | Inline element spacing within CTAs |
| md | 16px (p-4) | BlogCard internal padding (existing pattern) |
| lg | 24px (gap-6) | Grid column gap in RelatedArticles (md:grid-cols-3 gap-6) |
| xl | 32px (p-8) | CTA box internal padding (matches existing bottom CTA) |
| 2xl | 48px (mt-12 / mb-8) | Heading margin-bottom in RelatedArticles section |
| 3xl | 64px (mt-16) | Top margin before RelatedArticles section on blog post page |

Section padding: `section-spacing` utility class — `padding-block: var(--layout-section-padding-y)` (5rem / 80px). Applied to RelatedArticles `<section>` wrapper.

Container: `max-w-7xl mx-auto px-6 lg:px-8` — matches all existing marketing section patterns.

Exceptions: `print:hidden` applied to RelatedArticles `<section>` unconditionally (safe on compare pages, required on resource pages). No touch target exceptions — this phase has no interactive form inputs.

## Typography

All values derived from globals.css `@utility typography-*` definitions. Phase 37 introduces no new type sizes.

| Role | Size | Weight | Line Height | Utility / Class |
|------|------|--------|-------------|-----------------|
| Section heading | 20–24px fluid (--text-title-2) | 600 semibold | 1.35 (leading-snug) | `typography-h2` |
| CTA body text | 16px (--text-base) | 400 regular | 1.5 (leading-normal) | `text-muted-foreground` |
| CTA link text | 16px (--text-base) | 600 semibold | 1.5 | `font-semibold` |
| Card metadata | 12px (--text-xs) | 400 regular | 1.5 | `text-xs text-muted-foreground` |

Notes:
- RelatedArticles heading uses `typography-h2` — 20–24px fluid, semibold (600), line-height 1.35.
- BlogCard title uses `font-semibold leading-snug line-clamp-2` (existing component — do not modify).
- CTA link text uses `font-semibold` weight (600) — `text-primary` color already signals interactivity; the intermediate 500 weight adds noise without hierarchy benefit.
- No new font sizes introduced. Phase strictly reuses the established scale.

## Color

Tokens from globals.css `@theme` block (oklch, dark-mode aware via CSS variables).

| Role | Token | Resolved Light | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | `bg-background` | oklch(0.985 0.002 240) ≈ #F8F9FB | Page surface, article background |
| Secondary (30%) | `bg-card` / `bg-muted/30` | oklch(0.99 0.003 240) / muted tint | BlogCard surfaces, section tints |
| Accent (10%) | `text-primary` / `bg-primary/5` / `border-primary/20` | oklch(0.54 0.23 257) ≈ blue | Compare CTA box tint + link + border |
| Muted surface | `bg-muted` / `border-border` | oklch(0.94 0.01 240) | Resource CTA box (neutral, not blue) |
| Destructive | `--color-destructive` | oklch(0.577 0.245 25) ≈ red | Not used in this phase |

Accent reserved for (explicit list — nothing else):
1. Compare CTA box background tint (`bg-primary/5`)
2. Compare CTA box border (`border-primary/20`)
3. Compare CTA link text (`text-primary`)
4. Compare CTA link hover (`hover:text-primary/80`)
5. RelatedArticles section heading (`text-foreground` — not accent; heading is neutral)

Resource CTA box uses `bg-muted` + `border-border` (neutral muted palette) — deliberately NOT primary blue, to visually distinguish from compare CTA.

### RelatedArticles Layout

```
<section class="section-spacing print:hidden">
  <div class="max-w-7xl mx-auto px-6 lg:px-8">
    <h2 class="typography-h2 text-foreground mb-8">{title}</h2>
    <div class="grid md:grid-cols-3 gap-6">
      {posts.map(post => <BlogCard post={post} />)}
    </div>
  </div>
</section>
```

- Mobile: single column (stacked cards).
- Tablet (md: 768px+): 3-column grid with `gap-6` (24px).
- `BlogCard` component used as-is — no modifications to existing component.
- Returns `null` when no published posts match the slugs (no empty state rendered).

## Copywriting Contract

| Element | Copy |
|---------|------|
| RelatedArticles heading (compare page) | "Read the Full Comparison" |
| RelatedArticles heading (resource pages) | "Related Blog Posts" |
| Compare CTA body | "Compare TenantFlow vs {competitorName} side-by-side" |
| Compare CTA link | "See Full Comparison" |
| Resource CTA body | "Download our free related resource" |
| Resource CTA link | "View Resource" |
| Empty state | No empty state rendered — component returns `null` when no posts match. No user-facing empty message. |
| Error state | No error state rendered — component returns `null` on fetch failure (SSG build will not render unpublished posts). No user-facing error. |
| Destructive actions | None in this phase. |

Notes:
- `{competitorName}` is the human-readable name from `COMPETITORS[competitorSlug]?.name` (e.g. "Buildium", "AppFolio", "RentRedi").
- Copy is sentence-case throughout. No all-caps labels.
- `ArrowRight` icon (lucide-react, `size-4` / 16px) follows all CTA links inline (`ml-1.5`).

## Interaction Contracts

### Hover States

| Element | Rest | Hover | Transition |
|---------|------|-------|------------|
| BlogCard | border bg-card, no shadow | `-translate-y-1 shadow-lg` | `duration-200` (existing pattern) |
| Compare CTA link | `text-primary` | `text-primary/80` | `transition-colors` |
| Resource CTA link | `text-primary` | `text-primary/80` | `transition-colors` |

### Focus States

All links inherit shadcn/Radix focus-visible ring (`focus-visible:ring-2 ring-ring`). No custom focus overrides.

### Loading States

- RelatedArticles on Server Component pages: rendered at SSG build time — no client-side loading state needed.
- Blog post page CTAs: purely static lookup (no data fetch) — renders synchronously with the page.
- No skeleton loaders needed for this phase.

### Print Behavior

`print:hidden` class on every `<RelatedArticles>` `<section>` wrapper — unconditional. Ensures resource page print output is unaffected.

## Accessibility

- `<section>` element wraps RelatedArticles — no `aria-label` required (heading provides the label via h2).
- `ArrowRight` icons are decorative — mark `aria-hidden="true"` on the icon element.
- BlogCard links are full-card links — `link` role conveyed by `<a>` element, title text provides accessible name.
- No new interactive widgets. No keyboard management changes.
