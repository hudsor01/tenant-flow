<objective>
Fix SEC-02 and SEC-03 — two delivery-config gaps in the CSP and cache headers.

SEC-02: both CSP definitions ship `img-src 'self' data: blob:` with NO storage origin, NO `media-src`, and NO `frame-src`. Private-route media loads cross-origin from the storage host `https://<project>.supabase.co` (signed/public URLs): maintenance photos (`<img>`) + videos (`<video>` → media-src), inspection photos, document image previews (`<img>`) + PDF previews (`<iframe>` → frame-src), and the avatar (`<img>`) are all CSP-blocked on private routes. `connect-src` already uses the `*.supabase.co` wildcard. Extend BOTH `src/proxy.ts` `buildNonceCsp` and `vercel.json`: `img-src 'self' data: blob: https://*.supabase.co`; add `media-src 'self' data: blob: https://*.supabase.co`; add `frame-src 'self' https://*.supabase.co` (for the PDF iframe). Do NOT introduce a bare `https:`. Keep the two files' CSP strings in sync. (Property images are fine — next/image, same-origin — no change needed there.)

SEC-03: `vercel.json` sets `/properties/(.*)` → `Cache-Control: public, s-maxage=300, stale-while-revalidate=3600` + `Vary: Authorization, Cookie`. The Vercel CDN does NOT honor `Vary: Cookie` for cache-key separation, so owner A's property-detail HTML can be served to owner B within the 300s window — a cross-tenant data leak on an auth-walled route (`/properties` is in `PRIVATE_ROUTE_PREFIXES`). Replace that block with a single `Cache-Control: private, no-store, must-revalidate` header (defense-in-depth, auditable) and DELETE the dead `/manage` + `/tenant` exact-match blocks (no such routes exist; `/tenants` plural doesn't match `/tenant`). Preserve all legit caching (static assets, marketing, OG, blog, manifest, favicon, api no-store).

Purpose: let private-route storage media render, and stop an auth-walled route from being shared-cacheable.
Output: synchronized CSP directives in both files; a private no-store /properties rule; two dead rules removed.

NOTE (wave 2): this plan and 33-01 both edit `src/proxy.ts`, so it runs after 33-01. 33-01 changes the proxy() body (the aal gate); this plan changes only the `buildNonceCsp` directive list. Rebase onto 33-01's proxy.ts and touch only the CSP array.
</objective>
