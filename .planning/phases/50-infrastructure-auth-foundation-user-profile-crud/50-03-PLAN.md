Key DB schema facts:
- tenants table has: emergency_contact_name, emergency_contact_phone, emergency_contact_relationship columns
- Emergency contact reads/writes go to supabase.from('tenants') filtered by user_id (RLS enforces ownership)
- Sessions: Supabase exposes active sessions via supabase.auth.getSession() for the current session. Listing ALL sessions requires the Admin API or a custom Supabase function. The NestJS endpoint /api/v1/users/sessions returns sessions with device info. For the PostgREST path, the Supabase JS client exposes supabase.auth.admin.listUserSessions(userId) but this requires the service role key, which should NOT be used in the browser client.
- Alternative for sessions: use the Supabase REST Admin API via a Next.js API route (server-side) or fall back gracefully to returning only the current session when PostgREST is enabled.
