  <action>
Create `supabase/functions/_shared/resend.ts` with a `sendEmail` function that:

1. Reads `RESEND_API_KEY` from `Deno.env.get('RESEND_API_KEY')`.
2. Accepts params: `{ to: string[]; subject: string; html: string; tags?: Array<{ name: string; value: string }> }`.
3. POSTs to `https://api.resend.com/emails` with:
   - `Authorization: Bearer ${RESEND_API_KEY}` header
   - `Content-Type: application/json` header
   - Body: `{ from: 'TenantFlow <noreply@tenantflow.app>', to, subject, html, tags }` (omit tags if undefined)
4. Returns `{ success: true; id: string }` on 200, or `{ success: false; error: string }` on failure.
5. The function NEVER throws -- all errors are caught and returned as `{ success: false; error: string }`.
6. Logs errors with `console.error('[RESEND_ERROR] ...')` for Supabase log capture.
7. If `RESEND_API_KEY` is not set, return `{ success: false; error: 'RESEND_API_KEY not configured' }` immediately without making a fetch call.

Export types: `SendEmailParams` and `SendEmailResult`.

Follow the project convention from `_shared/cors.ts` -- simple utility functions, no classes, direct exports.

Do NOT use the Resend Node.js SDK. Use raw `fetch()` per Supabase Edge Function best practice.

**A) email-layout.tsx** -- Shared branded layout wrapper:
- Props: `{ children: React.ReactNode; previewText: string }`

- No reply-to disclaimer in footer

- Export as `EmailLayout`

**B) payment-receipt.tsx** -- Tenant receipt:
- Props interface `PaymentReceiptProps`:
  - `tenantName: string` -- greeting name
  - `amount: string` -- formatted total (e.g., "$1,200.00")
  - `propertyAddress: string` -- full property address
  - `unitNumber: string | null` -- unit number if applicable
  - `paymentDate: string` -- formatted date (e.g., "February 27, 2026")
  - `periodMonth: string` -- e.g., "March"
  - `periodYear: string` -- e.g., "2026"
  - `paymentMethodLast4: string | null` -- card last 4 digits
  - `lateFeeAmount: string | null` -- if late fee was included
  - `baseRentAmount: string | null` -- base rent when late fee present
- Preview text: "Payment Receipt - ${periodMonth} ${periodYear}"
- Wraps content in EmailLayout
- Shows "Payment Successful" heading (same messaging regardless of overdue status per user decision)
- Displays: greeting with tenant name, amount paid, property address + unit, payment date, period (month/year)
- If paymentMethodLast4 provided: "Paid with card ending in {last4}"
- If lateFeeAmount AND baseRentAmount provided: itemized breakdown table:
  - Base rent: {baseRentAmount}
  - Late fee: {lateFeeAmount}
  - Total paid: {amount}
- If no late fee: just show total amount
- Export as `PaymentReceipt`

**C) owner-notification.tsx** -- Owner notification:
- Props interface `OwnerNotificationProps`:
  - `ownerName: string` -- greeting name
  - `tenantName: string` -- who paid

  - `paymentDate: string` -- formatted date
- Preview text: "Payment received from ${tenantName}"
- Wraps content in EmailLayout
- Shows "Payment Received" heading
- Displays: greeting with owner name, tenant name who paid, amount received, property/unit, payment date
- Brief, informational -- "heads-up that money came in" per user decision
- Export as `OwnerNotification`
