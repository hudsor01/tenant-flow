1. **`errors.ts`** — Shared error response utility per CONTEXT.md locked decision:
   - Export `errorResponse(req: Request, status: number, error: unknown, context?: Record<string, unknown>): Response`
   - Logs full error details to Sentry (if SENTRY_DSN set) via `Sentry.captureException` with `extra` context including `req.url` and `req.method`
   - Also logs to `console.error` as structured JSON: `{ level: 'error', status, message, ...context }`
   - Returns generic JSON response: `{ error: 'An error occurred' }` with CORS headers from `getCorsHeaders(req)`

   - Export `validateEnv(config: { required: string[], optional?: string[] }): Record<string, string>`
   - Checks all required vars exist and are non-empty; throws Error listing missing vars if any are missing
   - Logs `console.warn` for optional vars that are unset
   - Uses module-level `let cachedEnv` for warm-isolate optimization (harmless if isolate is cold)
   - Returns typed config object with all found env vars keyed by their name
   - This is called inside `Deno.serve` handler on first request, NOT at module level

3. **`escape-html.ts`** — Extract escapeHtml from auth-email-templates.ts:

   - Escapes `&`, `<`, `>`, `"`, `'` (same implementation as auth-email-templates.ts line 10)
