**Implementation approach:**
- Supabase DB Webhooks at the SQL level use `pg_net` extension via `net.http_post()` to fire HTTP requests from PostgreSQL triggers.
- The `net` schema is available in Supabase projects — `pg_net` is enabled by default.
- Each trigger function is `SECURITY DEFINER` and reads webhook URL + secret from `current_setting()` — Supabase stores secrets as PostgreSQL configuration parameters accessible via `current_setting('app.settings.N8N_WEBHOOK_SECRET', true)`.
- Alternative approach: if `net.http_post()` is not available (Supabase sometimes uses `supabase_functions.http_request` instead), fall back to using `pg_net` directly. Use the `net.http_post()` function signature from pg_net:
  ```sql
  net.http_post(
    url text,
    body jsonb,
    headers jsonb default '{}'::jsonb,
    timeout_milliseconds integer default 5000
  ) returns bigint
  ```
- The trigger is AFTER INSERT/UPDATE so the `NEW` row is fully committed before the HTTP request.
- Webhook calls are fire-and-forget (the trigger function does not wait for the response). This is the correct pattern — retries are handled by Supabase DB Webhook built-in retry (via pg_net's async queue).

**Header comment** — explain the migration, note that pg_net must be enabled (it is by default on Supabase), and that secrets must be set before webhooks fire.

**Section 2 — rent_payments webhook trigger (WF-01):**

Fires on INSERT only (payment recorded). Sends the full new row as JSON body. Includes `type: 'INSERT'` field for n8n routing clarity.

```sql
create or replace function public.notify_n8n_rent_payment()
returns trigger
language plpgsql
security definer
set search_path = public, extensions
as $$
declare
  v_webhook_url text;
  v_webhook_secret text;
begin
  -- Read n8n configuration from Supabase app settings
  -- These must be set as Supabase secrets: N8N_WEBHOOK_RENT_PAYMENT_URL, N8N_WEBHOOK_SECRET
  v_webhook_url := current_setting('app.settings.N8N_WEBHOOK_RENT_PAYMENT_URL', true);
  v_webhook_secret := current_setting('app.settings.N8N_WEBHOOK_SECRET', true);

  -- Abort gracefully if URL not configured (do not fail the INSERT)
  if v_webhook_url is null or v_webhook_url = '' then
    return new;
  end if;

  -- Fire async HTTP POST to n8n — non-blocking
  perform net.http_post(
    url := v_webhook_url,
    body := jsonb_build_object(
      'type',   'INSERT',
      'table',  'rent_payments',
      'record', row_to_json(new)::jsonb
    ),
    headers := jsonb_build_object(
      'Content-Type',  'application/json',
      'Authorization', 'Bearer ' || v_webhook_secret
    ),
    timeout_milliseconds := 5000
  );

  return new;
end;
$$;

comment on function public.notify_n8n_rent_payment() is
  'Trigger function: fires on rent_payments INSERT and POSTs payment row to n8n. n8n workflow sends owner notification and generates receipt. Non-blocking — gracefully skips if N8N_WEBHOOK_RENT_PAYMENT_URL secret not set.';

-- Drop trigger if exists to allow re-run of migration
drop trigger if exists trg_rent_payments_notify_n8n on public.rent_payments;

create trigger trg_rent_payments_notify_n8n
  after insert on public.rent_payments
  for each row
  execute function public.notify_n8n_rent_payment();
```

**Section 3 — maintenance_requests webhook trigger (WF-02):**

Fires on INSERT (new request) and UPDATE (status change). For UPDATE, only fire if `status` actually changed (avoid firing on unrelated field updates).

```sql
create or replace function public.notify_n8n_maintenance()
returns trigger
language plpgsql
security definer
set search_path = public, extensions
as $$
declare
  v_webhook_url text;
  v_webhook_secret text;
  v_event_type text;
begin
  v_webhook_url := current_setting('app.settings.N8N_WEBHOOK_MAINTENANCE_URL', true);
  v_webhook_secret := current_setting('app.settings.N8N_WEBHOOK_SECRET', true);

  if v_webhook_url is null or v_webhook_url = '' then
    return new;
  end if;

  -- For UPDATE: only fire if status changed (avoid spurious webhook calls)
  if TG_OP = 'UPDATE' and old.status = new.status then
    return new;
  end if;

  v_event_type := TG_OP;  -- 'INSERT' or 'UPDATE'

  perform net.http_post(
    url := v_webhook_url,
    body := jsonb_build_object(
      'type',   v_event_type,
      'table',  'maintenance_requests',
      'record', row_to_json(new)::jsonb
    ),
    headers := jsonb_build_object(
      'Content-Type',  'application/json',
      'Authorization', 'Bearer ' || v_webhook_secret
    ),
    timeout_milliseconds := 5000
  );

  return new;
end;
$$;

comment on function public.notify_n8n_maintenance() is
  'Trigger function: fires on maintenance_requests INSERT (notify owner of new request) and UPDATE (notify tenant of status change). n8n branches on type field. Only fires on UPDATE when status field changes. Non-blocking.';

drop trigger if exists trg_maintenance_notify_n8n on public.maintenance_requests;

create trigger trg_maintenance_notify_n8n
  after insert or update on public.maintenance_requests
  for each row
  execute function public.notify_n8n_maintenance();
```
