
    **Step 2: Add `reactCompiler: true` to next.config.ts:**
    - Add `reactCompiler: true` as a top-level property on the nextConfig object (NOT under `experimental` -- it was promoted to stable in Next.js 16)
    - Place it before the `experimental` key for readability
    - The config should look like:
    ```typescript

      reactCompiler: true,

      images: { ... },
    }
    ```

    **Step 3: Verify build works:**
    ```bash
    pnpm typecheck && pnpm lint
    ```

    If the compiler encounters components that violate Rules of React and cannot be compiled, it will silently skip them (not fail the build). The existing `eslint-plugin-react-hooks@^7.0.1` catches these violations via lint.

    Note: Build time may increase modestly (10-30%) because the compiler uses Babel instead of SWC for relevant files. This is expected and acceptable per research findings.
