Rewrite the webhook authentication block in `docuseal-webhook/index.ts` to be fail-closed with HMAC signature verification. Per user decision: HMAC via `X-DocuSeal-Signature` header, return 401 for missing/invalid, no descriptive error messages.

**IMPORTANT:** After plan 58-01 runs, this file will have CORS headers removed and bare import specifiers. Work with the post-58-01 state. Do NOT re-add CORS headers.

**Step 1: Replace the auth block (currently lines ~228-236 in main handler).** The current pattern is:
```typescript
const webhookSecret = Deno.env.get('DOCUSEAL_WEBHOOK_SECRET')
if (webhookSecret) { /* verify */ } else { console.warn(...) /* continues processing */ }
```

Replace with fail-closed pattern:

1. Read `DOCUSEAL_WEBHOOK_SECRET` from env. If not set, log `console.error('DOCUSEAL_WEBHOOK_SECRET not configured')` and return `401 Unauthorized` immediately. This is the "fail-closed" behavior -- no processing if secret is missing.

2. Read `x-docuseal-signature` header (lowercase). If header is missing, return `401 Unauthorized`.

3. Read the request body as TEXT first (not JSON) with `req.text()`. This is critical -- HMAC verification needs the raw body bytes, and `req.json()` consumes the stream.

4. Verify HMAC-SHA256 signature using Web Crypto API:
   ```typescript
   const encoder = new TextEncoder()
   const key = await crypto.subtle.importKey(
     'raw',
     encoder.encode(webhookSecret),
     { name: 'HMAC', hash: 'SHA-256' },
     false,
     ['sign']
   )
   const sig = await crypto.subtle.sign('HMAC', key, encoder.encode(rawBody))
   const expectedSignature = Array.from(new Uint8Array(sig))
     .map(b => b.toString(16).padStart(2, '0'))
     .join('')
   ```

5. Compare signatures. Use a constant-time comparison to prevent timing attacks. In Deno, use `crypto.subtle.timingSafeEqual` if available, or compare the computed HMAC against the received signature by importing both into ArrayBuffers and comparing:
   ```typescript
   const sigBytes = encoder.encode(signature)
   const expectedBytes = encoder.encode(expectedSignature)
   if (sigBytes.byteLength !== expectedBytes.byteLength) {
     return new Response(JSON.stringify({ error: 'Unauthorized' }), { status: 401, headers: { 'Content-Type': 'application/json' } })
   }
   const match = crypto.subtle.timingSafeEqual(sigBytes, expectedBytes)
   if (!match) {
     return new Response(JSON.stringify({ error: 'Unauthorized' }), { status: 401, headers: { 'Content-Type': 'application/json' } })
   }
   ```
   Note: `crypto.subtle.timingSafeEqual` is available in Deno. If it is not available at runtime, fall back to comparing hex strings via `===` (acceptable for hex digest comparison since the digest itself prevents practical timing attacks).

6. Parse the verified body: `const body = JSON.parse(rawBody)` -- do NOT call `req.json()` since the body was already consumed by `req.text()`.

**Step 2: Remove the fallback `req.headers.get('authorization')` check.** The current code falls back to checking the `authorization` header as a bearer token. Per user decision, only `X-DocuSeal-Signature` HMAC is supported. Remove the `authorization` header fallback entirely.

**Step 3: Error responses must be generic.** All 401 responses must be `{ error: 'Unauthorized' }` with no additional details. No "Invalid signature", no "Missing header" -- just "Unauthorized". This is per user locked decision on error responses.

**Step 4: Remove the OPTIONS handler** if plan 58-01 has not already done so (it should have removed it since this is a webhook-only function).

**SEC-02: docuseal/index.ts -- Add ownership check after each lease fetch.**

The function has 5 actions that fetch a lease and operate on it: `send-for-signature`, `sign-owner`, `sign-tenant`, `cancel`, `resend`. Currently, after authenticating the JWT (`user` variable is available), each action fetches the lease by ID but never verifies `lease.owner_user_id === user.id`.

For each of the 5 action blocks, add an ownership check immediately after the lease fetch succeeds:

**For `send-for-signature` (already selects `owner_user_id` in its query):**
After the lease fetch and before the parallel profile/tenant/unit queries, add:
```typescript
if (lease.owner_user_id !== user.id) {
  return new Response(
    JSON.stringify({ error: 'Forbidden' }),
    { status: 403, headers: { ...getCorsHeaders(req), 'Content-Type': 'application/json' } }
  )
}
```

**For `sign-owner`:** The current `.select('id, docuseal_submission_id, tenant_signed_at')` does NOT include `owner_user_id`. Add `owner_user_id` to the select clause: `.select('id, owner_user_id, docuseal_submission_id, tenant_signed_at')`. Then add the same ownership check.

**For `sign-tenant`:** This is the ONE action where a tenant (not the owner) should be allowed. The current `.select('id, docuseal_submission_id, owner_signed_at')` does not include ownership info. Change the select to include both `owner_user_id` and `primary_tenant_id`: `.select('id, owner_user_id, primary_tenant_id, docuseal_submission_id, owner_signed_at')`.

For `sign-tenant`, the authorization check must allow EITHER the owner OR the lease's tenant:
```typescript
// Allow owner or the lease's primary tenant
const { data: tenantRecord } = await supabase
  .from('tenants')
  .select('id')
  .eq('user_id', user.id)
  .maybeSingle()

const isTenant = tenantRecord?.id === lease.primary_tenant_id
const isOwner = lease.owner_user_id === user.id

if (!isOwner && !isTenant) {
  return new Response(
    JSON.stringify({ error: 'Forbidden' }),
    { status: 403, headers: { ...getCorsHeaders(req), 'Content-Type': 'application/json' } }
  )
}
```

**For `cancel`:** Add `owner_user_id` to select: `.select('id, owner_user_id, docuseal_submission_id')`. Then add the ownership check.

**For `resend`:** Add `owner_user_id` to select: `.select('id, owner_user_id, docuseal_submission_id')`. Then add the ownership check.

**CRITICAL:** After lease fetch returns null/error, the current code returns `404 Lease not found`.
