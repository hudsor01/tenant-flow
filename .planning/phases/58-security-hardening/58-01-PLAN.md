**SEC-04: Fix notification_type CHECK constraint violation in stripe-webhooks/index.ts.**

The notifications table has a CHECK constraint: `notification_type in ('maintenance', 'lease', 'payment', 'system')`. Two inserts use values outside this constraint, causing CHECK violations and Stripe retry loops.

Map to existing constraint values (do NOT create a DB migration -- map to existing values instead):
- Line 170: Change `notification_type: 'stripe_connect_verified'` to `notification_type: 'system'` (it is a system-level connect verification event; the `title` and `message` fields already carry the specific context)
- Line 244: Change `notification_type: 'payment_failed'` to `notification_type: 'payment'` (it is a payment event; the `title` and `message` fields already carry the failure context)

**SEC-07: Create shared CORS helper at `supabase/functions/_shared/cors.ts`.**

Create the `_shared/` directory if it does not exist. Create `cors.ts` with two exported functions:

1. `getCorsHeaders(req: Request): Record<string, string>` -- reads `FRONTEND_URL` from `Deno.env.get()`. If origin header matches `FRONTEND_URL`, returns CORS headers with `Access-Control-Allow-Origin` set to the origin, plus `Access-Control-Allow-Headers: authorization, x-client-info, apikey, content-type`, `Access-Control-Allow-Methods: POST, GET, OPTIONS`, and `Vary: Origin`. If `FRONTEND_URL` is not set, log a warning and return empty headers (fail-open to avoid breaking all functions during config gap). If origin does not match, return empty headers.

2. `handleCorsOptions(req: Request): Response | null` -- if `req.method === 'OPTIONS'`, call `getCorsHeaders(req)`. If headers are non-empty, return `new Response('ok', { headers })`. If headers are empty, return `new Response(null, { status: 204 })`. If method is not OPTIONS, return null.

Classify each function and apply the appropriate CORS pattern:

**Browser-facing functions** (use shared CORS helper): `docuseal`, `generate-pdf`, `export-report`, `stripe-connect`, `stripe-billing-portal`, `stripe-checkout`, `stripe-checkout-session`, `tenant-invitation-validate`, `tenant-invitation-accept`

1. Add import: `import { getCorsHeaders, handleCorsOptions } from '../_shared/cors.ts'`
2. Remove the top-level `const corsHeaders = { ... }` block
3. Replace the OPTIONS handler: `const optionsResponse = handleCorsOptions(req); if (optionsResponse) return optionsResponse;`
4. For every `Response` that previously used `corsHeaders`, replace with `getCorsHeaders(req)`. Example: `{ ...corsHeaders, 'Content-Type': 'application/json' }` becomes `{ ...getCorsHeaders(req), 'Content-Type': 'application/json' }`. Note: you must pass the `req` variable, so ensure `req` is in scope at each usage point.

**Webhook-only functions** (remove CORS entirely): `stripe-webhooks`, `docuseal-webhook`

4. Do NOT import the shared CORS helper
