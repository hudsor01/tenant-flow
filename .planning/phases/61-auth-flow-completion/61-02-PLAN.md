# 61-02: Email Confirmation Flow

## Goal
After signup, users see a "check your email" page with a resend button that has a 60-second rate limit. Clicking the confirmation link in email activates the account and routes to the correct dashboard.

## Current State
- Confirm email page exists at `src/app/auth/confirm-email/page.tsx` with a full auth layout
- Resend button exists but has NO rate limiting (can spam resend)
- Login page already redirects to `/auth/confirm-email` when email is not confirmed (line 67-69)
- Signup mutation in `use-auth.ts` redirects to `/auth/confirm-email` when `!data.user?.confirmed_at`
- Auth callback route at `/auth/callback/route.ts` handles OAuth code exchange but NOT email confirmation tokens
- Supabase email confirmation uses PKCE flow: link has `token_hash` and `type=signup` params
- The callback needs to handle `type=signup` to exchange the confirmation token for a session

## Tasks

### Task 1: Add 60-second rate limit to resend button
**File**: `apps/frontend/src/app/auth/confirm-email/page.tsx`
- Add `cooldownSeconds` state that counts down from 60
- After successful resend, set `cooldownSeconds = 60` and start countdown
- Disable button during cooldown, show remaining seconds
- Show "Email sent!" feedback after successful resend
- Show error toast if resend fails
- Use `useEffect` with `setInterval` for countdown

### Task 2: Create email confirmation callback handler
**File**: `apps/frontend/src/app/auth/callback/route.ts`
- The existing callback only handles OAuth `code` parameter
- Supabase email confirmation links include `token_hash` and `type` params
- When `type=signup` and `token_hash` is present: call `supabase.auth.verifyOtp({ token_hash, type: 'signup' })`
- On success: read `user_type` from the session and redirect to the correct dashboard (TENANT -> /tenant, else -> /dashboard)
- On failure: redirect to `/auth/confirm-email?error=invalid_token`

### Task 3: Handle confirmation errors on confirm-email page
**File**: `apps/frontend/src/app/auth/confirm-email/page.tsx`
- Check for `error=invalid_token` search param
- If present, show an error message: "The confirmation link is invalid or has expired. Please request a new one."
- The resend button is already available for recovery

## Files Modified
- `apps/frontend/src/app/auth/confirm-email/page.tsx` - rate limit + error handling
- `apps/frontend/src/app/auth/callback/route.ts` - email confirmation token handling

## Acceptance Criteria
- After signup, user sees confirm email page
- Resend button works with 60-second cooldown displaying countdown
- Shows "Email sent!" after successful resend
- Clicking confirmation link in email auto-logs in and routes to correct dashboard
- Invalid/expired confirmation link shows error with resend option
- If user tries to login before confirming: redirected to confirm page
