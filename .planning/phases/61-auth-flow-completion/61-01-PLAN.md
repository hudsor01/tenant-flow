# 61-01: Password Reset Flow

## Goal
Complete the password reset flow end-to-end: user clicks "Forgot Password" on login page, receives email with reset link, clicks link, lands on reset page with the auth layout, enters new password, and is redirected to login with a success toast.

## Current State
- `ForgotPasswordModal` exists at `src/components/auth/forgot-password-modal.tsx` - sends reset email
- `useSupabasePasswordResetMutation` in `src/hooks/api/use-auth.ts` calls `resetPasswordForEmail` with `redirectTo: /auth/reset-password`
- `UpdatePasswordForm` component exists at `src/components/auth/update-password-form.tsx` - handles password update
- Page exists at `/auth/update-password/page.tsx` with full auth layout (split view with image)
- **BUG**: The redirect URL is `/auth/reset-password` but the page is at `/auth/update-password`

## Tasks

### Task 1: Fix redirect URL mismatch
**File**: `apps/frontend/src/hooks/api/use-auth.ts`
- Change `redirectTo` in `useSupabasePasswordResetMutation` from `/auth/reset-password` to `/auth/update-password`
- This ensures the email link routes to the existing page

### Task 2: Update UpdatePasswordForm for reset flow improvements
**File**: `apps/frontend/src/components/auth/update-password-form.tsx`
- Change minimum password length from 8 to 6 (matching Supabase config `minimum_password_length = 6`)
- After successful reset: redirect to `/login` (not `/`) with success toast "Password reset successful. Please log in."
- Use `router.push('/login')` instead of `router.push('/')`

### Task 3: Handle expired/invalid reset links
**File**: `apps/frontend/src/app/auth/update-password/page.tsx`
- The page needs to detect when the reset token is expired/invalid
- Convert to a client component that checks for error hash params from Supabase
- When Supabase detects an expired/invalid token, it appends `#error=...&error_description=...` to the URL
- Show an error state with "This link has expired or is invalid" message
- Include a button to request a new reset email (opens ForgotPasswordModal or links to /login)
- When valid token: show the existing `UpdatePasswordForm`

## Files Modified
- `apps/frontend/src/hooks/api/use-auth.ts` - fix redirectTo URL
- `apps/frontend/src/components/auth/update-password-form.tsx` - password length + redirect
- `apps/frontend/src/app/auth/update-password/page.tsx` - expired/invalid link handling

## Acceptance Criteria
- Clicking "Forgot Password" on login sends email with link to `/auth/update-password`
- Valid link shows the password reset form in auth layout
- Expired/invalid link shows error with recovery option
- After successful reset: redirect to /login with success toast
- Password minimum is 6 chars (Supabase default)
